# 联盟创世档案：制宪会议

**性质**：GAIA 联盟诞生的第一份档案  
**用途**：记录宪章与协议诞生的过程；确立「非一人决策」的创世正当性  
**终极愿景**：建立艺术家的理想秩序；见[《GAIA 成立理由书》](../06_解释与叙事/GAIA%20成立理由书（艺术家为何联合）.html)§〇  
**版本**：v1.0  
**依据**：成立理由书「艺术家联合」「共识高于裁决」；与联盟宪章序言配套

---

## 一、制宪会议：不是一人决策

GAIA 的宪章与协议，**不是 Founder 一人决断的产物**，  
而是经由**制宪会议**式过程诞生的——  
从创世期起，即有 IT 系统协助记录、结构化、草拟，  
Founder 提供方向与最终确认，系统协助形成可引用、可追溯的文本。

**制宪会议**的涵义：

| 要素 | 说明 |
|------|------|
| **过程优先** | 宪章生于有记录、可追溯的商议过程，而非一纸自上而下的宣告 |
| **IT 系统协助** | 从创世期起，AI/技术系统即参与：记录输入、结构化表达、草拟条款、提出警示；不替代决策，但使决策可被文档化、可被传承 |
| **非一人决策** | Founder 是发起者与最终确认者；系统是协助者；未来裁定者、门类理事会、艺术家代表可逐步加入商议 |
| **第一份档案** | 本档案及关联记录（输入映射表、更新日志、对话沉淀），构成联盟诞生的第一份可查证档案 |

---

## 二、创世期：制宪会议 Phase 0

当前阶段为**创世期（Phase 0）**：

| 参与方 | 角色 | 产出 |
|--------|------|------|
| **Founder** | 方向、决策、确认 | 联盟宪章、成立理由书、创意载体规范、分润逻辑、递归反哺细则、公益金治理、财务分配参数等核心文本的实质内容 |
| **IT 系统（AI 协助）** | 记录、结构化、草拟、警示 | 将输入转化为规范文本；维护索引与追溯；提出合宪性警示；不越权决策 |
| **协作约定** | 操作协议 | [《Founder-AI 协作约定》](Founder-AI%20协作约定.html)界定双方边界；[《输入与解释映射表》](输入与解释映射表.html)记录输入来源 |

**产出物的正当性**：  
虽创世期参与方有限，但因有**过程记录**（输入→含义→所在文件→解释位置），  
宪章与协议可被追溯至具体输入与决策时点；  
未来参与者可查阅「为何如此约定」，而非面对无法解释的既成文本。

| 档案组件 | 路径 | 内容 |

**产出物**（制宪会议产物）：  

| 类别 | 文档 |
|------|------|
| 宪章级 | GAIA 联盟宪章（序言、第一至六编、合章性裁定） |
| 解释性 | GAIA 成立理由书 |
| 协议级 | 创意载体规范、递归反哺与源头确权细则、CTO 技术共识、FCR、三项权力 |
| 工程级 | 分润与财务逻辑、财务分配参数设计依据、公益金治理 |

## 四、未来阶段：制宪会议扩展

当裁定者、门类理事会、艺术家代表等具备参与条件时，  
制宪会议可扩展为**多参与方商议**：

- 新参与者可提出修正案、补充条款
- 经共识程序（见联盟宪章第六编）采纳
- 商议过程继续由系统记录，追加至本档案
- 创世期档案永久保留，作为「联盟如何诞生」的可查证底本

---

## 五、与成立理由书的对应

| 成立理由书 | 制宪会议 |
|------------|----------|
| 艺术家联合，非一人决策 | 制宪会议从一开始即有过程、有记录 |
| 社会力量共同完成联盟 | 创世期 Founder + IT；未来裁定者、门类理事会等加入 |
| 规则被写入、被实施、可演进 | 制宪会议产出规则；系统协助使其可追溯、可修订 |

**本档案的确立**：  
联盟的诞生，从第一天起即有「制宪会议」式的过程与记录；  
IT 系统协助宪章的诞生，使这一过程成为联盟的第一份可传承档案。

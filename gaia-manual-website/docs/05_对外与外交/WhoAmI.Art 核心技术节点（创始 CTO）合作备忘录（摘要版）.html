WhoAmI.Art 核心技术节点（创始 CTO）合作备忘录（摘要版）

签署双方：

  • 甲方（执行发起人/Founder）： [姓名]
  • 乙方（技术发起人/创始 CTO）： [姓名]

立约背景： 双方基于对「新商业文明」及「联邦自治治理」的共识，决定共同铸造 WhoAmI.Art 系统。乙方作为 Batch 0（创始批次） 的唯一技术核心，负责将本联邦的治理意志转化为不可篡改的代码逻辑。

开篇：三句话核心逻辑（制度智慧对齐技术价值）

以下三句话，是 Founder 与 IT 伙伴达成共识的心理锚点，也是本备忘录的制度内核：

维度 核心表述
安全 咱们不玩「谁控制谁」的权力游戏；我把所有分配规则写死在系统里，让代码成为保护你我利益的「宪法」，谁也赖不掉，包括我也不能。
身价 你写的不是随时能被外包替代的代码，而是作为 Batch 0 创始人的永久信用底座；技术会折旧，但你在这个联邦里占住的「第一代位置」和那份全局分润是永久增值的。
未来 咱们现在当 CEO 和 CTO 是为了让这个「机器」跑起来；等系统壮大了，咱们就请最牛的人来接管执行，咱们退到幕后拿着最干净的红利，去干真正想干的事。

逻辑拆解（Founder 宣讲用）：第一句破「疑」（放弃人治，钱交给系统）；第二句破「价」（入股规则，非干活）；第三句破「局」（禅让胸怀，托付信任）。

第一条：代码即宪法（安全保障）——对应「安全」

1.1 参数化分配：乙方确认并负责实现系统核心分配引擎，严格锁定比例：艺术家利润部分 65%、创始池 5%、运营池 20%、风险基金 10%。该比例一经上线，非经联邦宪政程序不得人工干预。

1.2 治理日志（Governance Log）：乙方的所有核心贡献及系统重大变更必须记录于 Append-only（只增不减）的治理日志中，确保每一分钱的去向均可追溯、可审计。

1.3 解耦原则：甲方承诺不干涉乙方对技术架构的专业实现；乙方承诺技术实现必须服从于「分配透明」与「艺术家主权」的宪法级约束。

1.4 入口主权：WHOAMI.ART 域名及核心品牌由甲方个人持有，以「信用质押物」形式授权给执行节点；乙方管逻辑，甲方管主权。公司可换、主权不动，确保无人能劫持。详见《域名主权与制宪者谈话》。

第二条：Batch 0 身份与永久权益——对应「身价」

2.1 创始节点地位：乙方不仅是代码开发者，更是系统的 Batch 0 创始节点。其贡献记录将永久硬编码于系统信用底座中。

2.2 全局分润权：乙方享有 5% Anchor Pool（创始池） 的分配权重。该权益与具体职位脱钩，只要系统存在，乙方即享有基于该权重的系统级红利。

2.3 衍生业务优先权：在 WhoAmI.Art 联邦扩张过程中产生的衍生业务（含：履约、供应链、合规结算、法务承包、财务承包、IT 承包、企业定制等，详见《衍生业务优先权与最惠国条款》），乙方在同等条件下享有优先承接权与投资权；须遵守最惠国条款及交叉审计。

第三条：职业化接管与长期自由——对应「未来」

3.1 职位折旧与接班：双方一致认可 CEO 与 CTO 职能随系统规模扩大而产生「管理折旧」。当系统日订单稳定超过 200 单 后，双方有权动议引入更具规模化能力的职业经理人团队接管日常执行。

3.2 退出不退利：执行权移交后，乙方作为「制宪者」保留其 Batch 0 级分润权及对 System Accumulation Pool（系统积累池） 的分配动议权。

3.3 禅让激励：乙方若在任期内成功培养并交接给下一代合格技术架构师，系统将从积累池中拨付专项「传承奖金」。

第四条：风险管理与退出机制

4.1 14 天争议缓冲：所有分润结算须经过 14 天 Escrow 周期，以对抗全球支付风险，确保系统财务安全。

4.2 Kill Switch：乙方承诺保留艺术家对自身授权的「一键撤回权」，这是系统获取顶级供给的核心信用保证。

4.3 技术审计：若发生重大治理冲突，乙方同意接受由起草委员会发起的第三方技术审计，以透明度换取共识。

附录：商人视角逻辑拆解（Founder 宣讲参考）

句子 破什么 逻辑
安全 破「疑」 针对高学历技术人的清高与防御。你主动放弃「人治」，把钱交给系统自动执行，比任何承诺都有力。
身价 破「价」 针对对自己技术劳动力的定位。他在系统里的价值不是「干活」，而是「入股」了整套规则的初始节点。
未来 破「局」 针对创业者常见的「权力执念」。提出禅让和职业化接管，展现胸怀与对财务自由的终极理解，让他觉得你可托付。

GAIA 节点角色与 IT 平台映射

用途：明确系统中各节点角色类型，及其与美国宪政体系部门的类比；规划 GAIA 官网及多 IT 平台下的功能映射
版本：v0.1
性质：架构设计文档，供官网与系统规划参考

本架构不仅是商业计划，更是数字联邦的制宪大纲。
终极目标是建立艺术家的理想国；十二类节点的设计，是实现这一目标的权力制衡与责任分摊结构。
在商人眼里，这是极高的社会资本组织艺术；在会计师眼里，这是清晰的责任与风险分摊模型。
通过将「灵魂（GAIA 规矩与档案）」与「肉体（商业实例）」分离，架构层面已实现永续经营（Going Concern）——即便某一实例因不可抗力消失，协议与档案依然存在。

一、结论先行：GAIA 需要官网

GAIA 须有一官网，作为：

  • 宪法、成立理由书、创世档案的正式发布与存证场所
  • 各节点按身份访问不同功能的统一入口（或入口索引）
  • 与 signa.ink、WhoAmI.Art 等商业实例区分：官网 = 规矩与档案；实例 = 商业层

二、节点主要角色类型

角色类型 定义 功能身份 类比（美国宪政）
创世者（Founder） 元规则制定者；三项不可谈判权力 执行权；72h 紧急暂停、最终解释权 总统（Executive）+ 制宪会议召集人
门类理事会（Domain Council） 按艺术门类设立；3–5 人；行业准入、竞争协调、伦理边界 准入权；不碰钱、不碰码 国会各专业委员会（Legislative Committee）
裁定者（Steward） 高贡献、参与共识裁定的专业节点 裁定权；陪审团；无永久法官 陪审团 + 大法官（Judicial）
分配引擎 参数化、自动执行、无人干预 分配权；按 RuleID 分账 国库/美联储（非民选，按规则运行）
执行节点（LLC） 代收代付、技术结算与合规接口 托管与结算；不拥有规则权 财政部/IRS（执行层，非决策层）
CTO / 技术节点 代码宪法执行者；分配引擎实现者；负责落实任何节点提出的 IT 建设需求 技术实现；无权改参数；保障联盟与时俱进、高效先进 技术官僚机构
创世公民 / Anchor Artist 创世公民；信用底座；签自然人宪章 5% Anchor 分润；Kill Switch 制宪会议代表 / 开国元勋
源头艺术家 拥有资产发行权的创作者 65% 分润；递归反哺接收；署名权 公民（财产权主体）
普通艺术家（Batch 1+） 标准化 DIL 签约；系统使用者 65% 分润；符合准入后可参与 公民（有限权）
用户 / 收藏者 购买身份授权、支持源头 支付、查阅、评价 公民（需求方）
SOC（公益节点） 经认证公益/社会组织 反哺接收方；可审计 慈善/非营利部门
营销节点 / 社会资本节点 引入流量、背书 CPS 分润；入口筛选建议 游说团体 / 利益相关方（受约束）
制宪会议 / 创世档案 过程记录；IT 系统协助 记录、结构化、可追溯 制宪会议记录 / 国家档案馆

2.1 权力制衡与多视角穿透

角色 商人视角的潜台词 会计师视角的防线
创世者（Founder） 72h 暂停是终极熔断机制，确保系统不在意外中崩塌 最终解释权即规则的会计准则制定者；掌握定义「利润」的话语权
门类理事会（Domain Council） 行业准入制造稀缺性；老兵社会资本实现行业自治 风险前置；同行审核质量，降低后续退货损失（FCR）
裁定者（Steward） 纠纷仲裁外包给共识，实现名誉风险隔离 每笔非标支出有证据链支撑，非个人意愿
创世公民 早期信用背书；利益共同体，非松散节点 信用底座即平台初始商誉（Goodwill），不可估量
SOC（公益节点） 占据道德高地；社会价值二次分配 合法税务抵扣通道；反哺是品牌抵御舆论的防火墙

2.2 灵肉分离：GAIA 与商业实例的双层主权

层级 性质 商人逻辑 会计逻辑
GAIA（灵魂层/中枢） 协议、宪法、创世档案；法律与档案库 核心资产隔离池；不参与买卖，故「洁净」；即便 signa.ink 因监管或攻击消失，GAIA 依然存在 永续经营基础；规矩与档案 与 风险敞口分离
signa.ink / WhoAmI.Art（肉体层/实例） 赚钱机器；对接 TikTok、Stripe、物流 商业执行层；可替换 风险敞口层；若产生税务纠纷，可经 GAIA 协议重启新实例，不丢失艺术家与用户数据

GAIA 官网不需要花哨，需要的是公信力。它是这个联邦的「国库」与「最高法院」——让所有人看到规则不可篡改、财务阳光透明。

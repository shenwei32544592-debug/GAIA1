## 一、核心名词大白话

| 术语 | 财务定义 | 大白话 |
|------|----------|--------|
| Gross Price | 用户支付总金额 | 客户刷卡的总钱数 |
| Hard Costs | 快递、第三方物料、支付手续费 | 必须给别人的钱，不参与分润 |
| Net Pool | 总价 - 硬成本 | 咱们自己人分得着的钱 |
| 艺术家利润部分 (65%) | 劳务、IP 授权、个人溢价 | 艺术家凭本事拿走的大头；创意价值回归源头 |
| 创世分润池 (2%+2%+1%) | 创世者 2%、创世技术官 2%、公益积累金 1% | 创世节点信用抵押；原始内核人物、必不可少守夜人角色；道义与结构性双重正当性；封死未来无限加码；1% 入[公益金池（全平台公益积累金）](../02_协议与规范/公益金池细则.html)；取之于民，用之于民 |
| 公共运营池 (20%) | 联盟运转成本 | IT、合规、客服、获客、溯源与分配引擎；先覆盖运营成本，**结余**按 FCR 比例分配；取之于民，用之于民 |
| 风险储备金 (10%) | 全体系保险池 | 退款、争议、FX 缓冲；达阈值后停计、超额转入 20%；结余可优先用于节点内人道救助；取之于民，用之于民 |

**财政结构原则**：65%、2%+2%+1%、20%、10% 不是「谁分走了」，而是这个联盟运转所必须的财政结构。类比宪政国家：没有法院、税务局、储备金，公民权利无法实现。艺术家联盟亦然。人民须看好钱袋子；所有池均有明确用途边界、可查、可审计。

### 1.1 各池用途与溢出规则总表

| 池子 | 比例 | 首要用途 | 结余/溢出去向 |
|------|------|----------|---------------|
| 艺术家利润部分 | 65% | 劳务、IP 授权、个人溢价；创意价值回归源头 | 直接分配，无结余 |
| 创世者分润 | 2% | Founder 信用抵押与制度设计对价；原始内核人物、必不可少守夜人角色；道义与结构性双重正当性 | 无溢出（固定 2%） |
| 创世技术官分润 | 2% | 技术架构与基础设施对价；原始内核人物、必不可少守夜人角色；道义与结构性双重正当性 | 无溢出（固定 2%） |
| 公益积累金 | 1% | [公益金池（全平台公益积累金）](../02_协议与规范/公益金池细则.html)（传承、教育、扶持新人） | 直接入公益金池 |
| 公共运营池 | 20% | IT、合规、客服、获客、溯源与分配引擎 | 结余 → FCR 持有者分配 |
| 风险储备金 | 10% | 退款垫付、争议解决、FX 缓冲、硬成本超支 | 达阈值后超额 → 转入 20%；池内结余可优先节点内人道救助；20%+10% 仍不足时可触发紧急状态 |
| 行业始祖待认领池 | 0.1%×订单 | 无指认源头时按品类暂存 | 找到始祖 → 付其账户；公投认定无法指认 → 转入公益 |
| 紧急状态 | 例外 | 20%+10% 仍不足时 | 14 天披露后 75% 共识；7 天生效；≤6 月；续延≤3 月；12 月冷却；60 天内事后审查 |

**公益金池来源**：1%（二人各 0.5% 合成）+ 行业始祖 0.1%（先入待认领池，经公投转入后入公益金池）+ 主动捐赠。详见[《公益金池细则》](../02_协议与规范/公益金池细则.html)。创世人个人捐不捐为个人选择，不写入协议。

### 1.2 建国信用池（Batch 1+ 艺术家限期回补）

**条件触发**：仅当订单来自 **Batch 1+ 艺术家**（建国成功或 FCR 窗口关闭后加入），且处于其**回补期内**（如前 90 天或前 30 单）时，该艺术家 Artist Pool 的 1%–3% 进入**建国信用池**。详见[《建国回补与确定性溢价机制（草案）》](../02_协议与规范/建国回补与确定性溢价机制（草案）.md)。

| 项目 | 约定 |
|------|------|
| **来源** | 艺术家利润部分（65%）中 carve-out 1%–3%；仅 Batch 1+ 回补期内 |
| **去向** | 建国信用池；按 FCR 权重分配予创世公民；不得进私人或公司 |
| **期限** | 回补为限期机制；满足 90 天或 30 单后自动停止；不构成永久抽成 |
| **可审计** | 池子流水与分配可追溯、可公示 |

**公平逻辑**：早期的人承担的是「系统可能失败」的风险；后来的人享受的是「系统已被证明可用」的确定性。后来者用一小段时间对早期试错成本做有限回补，这是同一套公平。

## 二、分配公式：穿透式结算

```
收钱：用户付 $1000
剥离：快递 $50 + 物料 $50 → 硬成本 $100
Net Pool：$900

分润（65/5/20/10，其中 5% = 2%+2%+1%）：
艺术家：$900 × 65% = $585
创世分润池：$900 × 5% = $45（含 Founder 2%、CTO 2%、公益 1%）
公共运营池：$900 × 20% = $180
风险储备金：$900 × 10% = $90
```

## 三、物料成本：Material Baseline（MB）

### 3.1 问题

艺术家自担大量物料成本（宣纸、特制颜料、贵金属）时，若死扣 65%，其纯利被压缩。

### 3.2 方案：先报销，后分红

| 原逻辑 | 新逻辑 |
|--------|--------|
| (总价 - 快递) × 65% = 艺术家 | (总价 - 快递 - 物料) × 65% + 物料 = 艺术家 |

物料从「分润」变为「报销」，从 Net Pool 计算前剥离。

### 3.3 平滑方式

| 方式 | 说明 |
|------|------|
| **分级 SKU** | 标准版 $499 含 $20 物料；典藏版 $799 含 $150 物料；系统自动识别 |
| **Class A 实报实销** | 高度定制时，艺术家上传凭证，平台先划拨物料费，不计入 Net Pool |
| **平台采购** | 由美国节点采购底座/框，艺术家只交付数字或轻量素材 |

## 四、最低利润率（MMG）

### 4.1 定义

**Net Pool / Gross ≥ 40%**（或单笔 Net Pool ≥ $200）。

低于此线，系统**不接单**。

### 4.2 意义

- 筛选高价值供给
- 保护 20% 公共运营池覆盖能力
- IT 只需：`IF (Price - Hard_Costs) / Price < 40% THEN REJECT_ORDER`

### 4.3 对外表述

「尊严红线」——拉低档次的不接。

## 五、20% 公共运营池：用途与结余分配

### 5.1 必须覆盖的成本（优先支出）

| 类别 | 内容 |
|------|------|
| **IT 与基础设施** | 服务器、API、数字存证 |
| **人工与客服** | 美国侧运营、纠纷处理 |
| **治理与合规** | LLC 年审、CPA、法律 |
| **品牌与获客** | 域名、SEO、策展 |

### 5.2 无人受雇原则（No-Employment Principle）

**联盟及其执行节点不与任何自然人建立劳动合同或固定雇佣关系。** IT、合规、客服、获客、治理支持等均以**外包**（按项目、按单、按量计费）或**选举岗位**（可设 per-task 补偿，非工资）形式运作。法定签字人由选举或指定产生，可为**多人协作签署**；其身份为受托人，非雇员。

**目的**：使 20% 池成本随业务量波动，无固定人力 overhead；保护 FCR 结余可预期性。无订单则无外包支出，结余自然显现。

### 5.3 结余分配：创始贡献者权利（FCR）

运营成本覆盖后的**结余部分**，按《FCR 授予书》约定比例分配给 90 日窗口内的创始贡献者。结余披露、审计要求见治理约定。

**全外包策略**：客服、快递、财务、IT、合规均外包；20% 用来「买服务」而非「养人」。按单计费，无订单无支出。此池为联盟运转所必须——无此，艺术家拿不到 65%，用户也买不到；取之于民，用之于民。

**创世后加入者**：创世成功后再加入的 IT、设计师、运营等，**不参与** FCR 分配；仅能通过工资或公民权（如艺术家 65%）创收。详见《FCR 附录》§4.2。

**供应商定价与 FCR 贡献**：第三方供应商（装裱、制作等）定价采用**可评估成本 + 合理利润**；供应商若为 FCR 持有者，其贡献以履约质量与长期合作为主，非价低者得。详见《分配方案：公式化定档与供应商定价》§第二条。

## 六、风险储备金处理

**术语说明**：风险储备金（Risk Reserve）与创始贡献者权利（FCR）为不同概念；本节仅指风险储备金。

### 6.1 存放

Mercury 逻辑隔离子账户；100% 流动性；利息归系统积累池。

### 6.2 首要用途（调用场景）

| 场景 | 触发 |
|------|------|
| 用户拒付/退款 | 自动从风险储备金垫付 |
| 硬成本超支 | 快递暴涨、供应商加价 |
| 汇率剧烈波动 | FX Buffer |

### 6.3 动态上限与溢出

| 规则 | 说明 |
|------|------|
| Cap | 过去 6 个月最高月 GMV 的 120%，或 Escrow 总量 20% |
| 达阈值后 | 不再计提 10%；后续本应入池的 10% **转入 20% 公共运营池** |
| 池内结余 | 在保证风控需求前提下，可优先用于**节点内人道主义救助**；决策程序由治理约定 |

### 6.4 目的表述

风险储备金为全体系保险池：保护艺术家与用户双方；达阈值后溢出回流联盟运转；结余可体现共同体关怀。取之于民，用之于民。

### 6.5 紧急状态：20% 与 10% 仍不足时的安全阀

当 20% 公共运营池不足以维持联盟运转，且已依规动用 10% 风险储备金后仍存在可预见的运营缺口时，可启动**紧急状态分配调整**。

| 环节 | 约定 |
|------|------|
| **触发** | 20% 耗尽 + 10% 风险金动用 + 仍有可预见运营缺口 |
| **提议** | 任一主体（创世公民 / 门类理事会 / 裁定者）可单独提议；须附**书面缺口说明与预算** |
| **披露期** | 提议后 **14 天** 披露期，方可表决；全平台可查 |
| **决策** | 创世公民 + 门类理事会 + 裁定者 **75% 以上共识**通过 |
| **生效延迟** | 通过后 **7 天** 生效；期间任何人可提请裁定者复核 |
| **措施** | 从艺术家利润部分（65%）**临时**提取不超过 5% 份额（具体比例及上限由动议约定），补入公共运营池 |
| **期限** | 每次有效期不超过 6 个月；续延须重新投票，**单次续延不超过 3 个月** |
| **冷却期** | 状态结束后 **12 个月内**不得再次启动 |
| **恢复** | 状态结束后，分配比例**自动恢复**为 65/5/20/10 |
| **事后审查** | 结束后 **60 天内**须公布：缺口成因、动用金额、恢复情况；裁定者须出具「是否必要」结论；全平台可查 |

**类比**：平时税率够用；战时须加税应对。此机制为非常状态下的安全阀，非对 65% 的永久削减。

## 七、第三方物料协作的穿透结算

当艺术家上架产品且使用第三方装裱/制作：

1. **上架**：勾选协作模式，锁定 Total Hard Costs（如 $200）
2. **支付**：用户付 $1000 → Stripe → 14 天 Escrow
3. **剥离**：$200 划入应付账款池，不参与分润
4. **Net Pool**：$800，按 65/5/20/10 分配
5. **结算**：14 天后自动给供应商 $150、物流 $50、艺术家 $520 等

**RuleID 要求**：OrderID 与 SupplierID 强绑定；Hard_Cost 零级优先隔离。

## 八、创世分润池（2%+2%+1%）对外表述

**结构**：创世者 2%、创世技术官 2%、公益积累金 1%；合计 5%，对外可称「创世信用与公益池」。**正当性**：道义（早期投入与风险回报）+ 结构性（二人为原始内核人物、必不可少守夜人，协议须其在场方能运转）。**过渡期**：当前为**创世守夜人过渡期**——此结构是到达系统完全安全平稳过程中的风险抵御机制；待更多守夜人加入、**风险解除**后，宪政共和国将实现自动安全运行，对所有人的保护由制度承担。风险解除条件与**创世荣誉监护**见术语表。

| 避免 | 推荐 |
|------|------|
| 建"国"功劳费、创始人红利 | **创世分润池**（联盟信用底座；2%+2%+1%） |
| — | **取之于民，用之于民**：创世节点信用抵押；封死未来无限加码；1% 入公益金 |
| — | **低费率裁定者（Low-Fee Guardian）** |

「创始节点通过固定 2%+2%+1% 封死未来无限加码抽成的可能。此结构是联盟运转必须的财政安排，不是谁分走了。」

### 8.1 创世节点收益之收款主体指定

为确保平台运营之财务独立性，并与创世者个人流水隔离，创世节点（创世公民 / Founder / 创世技术官）之 2% 制度对价收益遵循以下清算原则：

| 条款 | 内容 |
|------|------|
| **收款主体** | 创世节点有权指定其收益的接收主体；该主体可为本人、其控制的法律实体（LLC/Corp）、或专项事务信托（Trust） |
| **财务隔离** | 指定旨在将「规则对价」与个人日常流水分离，维护 GAIA 账本可审计性 |
| **效力延续** | 除非创世节点另行书面指示，系统沿用该指定路径；分配引擎按 RuleID 执行，无人干预 |

**依据**：执行节点（LLC）按创世者/CTO 书面指定的收款账户划付 2%；变更须经书面更新并通知财务。

## 九、1% 名誉成本

创始人各认捐 0.5%（合计 1%），预扣划入**公益金池**（与行业始祖基金同一账户）。自动划转、可抵税。**用途提议权与最终决策权**：该部分归属「创世者（Founder）决策权」，**创世者专属**；可主动提议用途亦可最终决定；创世技术官有知情权与建议权，无决定权；其它节点有提议权与监督权。动用前须提前 7 天向 CTO 披露；拨款决议须注明是否采纳 CTO 建议及理由。此为创世者制度性影响力来源，系其核心优势。详见《递归反哺与源头确权细则》§5.8.2、§5.8.2.1a。

**创世人个人捐赠**：创世人若自愿从个人所得中捐入公益金，为个人选择，不写入协议，不形成制度性差异。

## 十、递归反哺（收益回流）

**依据**：宪法第四编；《递归反哺与源头确权细则》

### 10.1 原则

- 回流必须从**经认定**的、在平台销售的、使用源头创意的作品收益中拆分
- 从衍生创作者的 **65%** 中抽取，由使用者让渡给源头
- 自认定日起执行

### 10.2 源头两层（最多两层，不越级）

| 层级 | 说明 |
|------|------|
| **绝对源头** | 真正原创者，可经审查动议事后确认 |
| **应用层源头** | 在绝对源头影响下创作、首次在平台登记的版权主体 |

### 10.3 定价权

- 版权使用费由**版权所有者**自主定价
- 平台不代为定价；提供定价入口与执行

### 10.4 建设费用

- 回流给艺术家的钱：来自衍生销售本身
- 回流机制（溯源、分配引擎）建设：来自公共运营池 20%

### 10.5 行业始祖 0.1%（不可指认源头）

当衍生作品声明「归属某品类传统，无具体可指认源头」时：

| 项目 | 规则 |
|------|------|
| **金额** | 65% 艺术家利润部分 的 0.1%（暂定比例；以后根据众人讨论后共识再明确确定），与协议默认一致 |
| **流向** | **行业始祖待认领池**（按品类暂存）；**不直接入公益** |
| **指认** | 经公众指认（PAC）或等效程序找到始祖 → 支付该品类历史积累及未来 0.1% 至其账户 |
| **转入公益** | 仅当公投认定无法指认或满 N 年无指认时，该品类余额转入公益金池 |

**详细规则**见《递归反哺与源头确权细则》§5 行业始祖 0.1%：待认领池与处分；池内动用权分层见 §5.8.2.1。

## 十一、财务分配参数：决策权与可变更性

### 11.1 创始人锁定（任何人无权染指）

以下参数由**创始人决策设定**，未经创始人书面同意，任何人、任何治理主体不得变更。执行节点、门类理事会、裁定者均无权修改。

| 参数 | 当前值 | 说明 |
|------|--------|------|
| 主分配比例 | 65 / 5 / 20 / 10 | Artist / 创世分润池(2%+2%+1%) / 公共运营池 / 风险储备金；硬编码，执行节点无权改 |
| MMG（最低利润率） | Net Pool / Gross ≥ 40% | 尊严红线；低于此线不接单 |
| 创世分润池 | Net Pool 的 5%（2%+2%+1%） | Founder 2%、CTO 2%、公益 1% |
| 创世公民 Cap | 单人单月 ≤ 过去 6 个月平均的 200% | 超出部分 Catch-up |
| Catch-up 分流 | 50% → 创始贡献者池，50% → 公共运营池 | 超出 Cap 部分的去向 |
| Time Decay | 18 个月线性衰减至 50% | 创世公民 时间衰减 |
| 订单触发衰减 | 1k 单 -10%，10k 单再 -10% | 创世公民 量级衰减 |
| 风险储备金比例 | 10%（含 1% FX Buffer） | 全体系保险池；达阈值后溢出转入 20% |
| 1% 名誉成本 | 创始人各 0.5%，合计 1% | 创始人认捐，流入公益金池 |
| 公益金池创世者决策权 | 1% 名誉成本的提议权与用途决策权；**创世者（Founder）专属**；创世技术官知情权与建议权、无决定权；其它节点提议权与监督权 | 受 §5.8.2.2–5.8.2.4 边界约束 |

**变更程序**：上述参数若需变更，须经**创始人书面同意**，并视条款级别同步履行：多签（Founder + CTO + 法务/财务）、或共识程序（见下）。

### 11.2 可变更参数：形式与程度

以下参数可经规定程序变更，变更权不在创始人独家持有。

| 参数 | 当前值 | 变更形式 | 变更程度 | 决策/参与主体 |
|------|--------|----------|----------|---------------|
| 递归反哺默认比例 | 65% 的 0.1% | 共识程序 | 可调；总回流封顶不变（≤65% 的 50%） | 经裁定者/共识程序修订 |
| 多源引用封顶 | 65% 的 50% | 共识程序 | 固定为 50%；不得突破封顶逻辑 | 裁定者裁定 |
| 宪法第四编收益比例 | 见宪法 | 第六编多阶段共识 | 须 75% 裁定者同意 + 90 天缓冲 | 宪法修正程序 |
| 公共运营池 20% 内部切分 | 运营/IT/合规/获客 | 运营层决策 | 内部调配，不改变 20% 总额 | Founder 或指定运营负责人 |
| 营销节点 CPS 比例 | 10% 公共运营池给营销 |  Domain Council 建议 | 在 20% 公共运营池内调剂 | Domain Council 采纳后执行 |

| 新人准入评分权重 | 艺术 40%/授权 20%/交付 20%/生态 20% | Domain Council 建议 + Founder 采纳 | 权重可调；≥75 通过线可经共识调整 | Domain Council 提议，Founder 合规复核 |

### 11.3 变更程序层级

| 层级 | 适用对象 | 程序 |
|------|----------|------|
| 创始人书面同意 | §11.1 所列参数 | 创始人签字；视需同步 Founder + CTO + 法务/财务 多签 |
| 共识程序 | 递归反哺默认比例、多源封顶、FCR 细则等 | 裁定者超级多数（≥75%）+ 披露期 |
| 宪法修正 | 第四编收益与反哺等宪法条款 | 第六编：技术审查 + 伦理评审 + 裁定者 75% + 90 天缓冲 |

### 11.4 不可篡改

无论何种程序，**RuleID 数据库生成的分配指令**为最高执行依据；会计与结算系统须服从 RuleID，任何人不得 override 已生效的分配结果。

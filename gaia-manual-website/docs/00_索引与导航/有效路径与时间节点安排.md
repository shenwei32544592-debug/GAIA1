**版本**：v1.1  
**用途**：确保最终目标实现时，具备有效路径与清晰的时间节点；可追踪、可复盘；与《建国路线图》《建国成功判定速查表》《创世贡献记录与确认机制》配套  
**最后修订**：2025-02

## 一、最终目标（North Star）

**终极愿景**：建立**艺术家的理想秩序**——创意主权被尊重，源头被铭记，收益可回流，共识有力量。  
这不是乌托邦；我们有分步、有目标、清晰的战略规划。美国建国时无人相信会成立，但通过制宪、分权、一步步落实，它实现了。我们亦然。见[《GAIA 成立理由书》](../06_解释与叙事/GAIA%20成立理由书（艺术家为何联合）.md)§〇。

| 层级 | 目标 | 检验标准 |
|------|------|----------|
| **文明层** | GAIA 协议在现实中「已发生、可签署、可验证、不可否认」 | 宪法与创意载体规范冻结；001 号物证存在 |
| **商业层** | WhoAmI.Art（GAIA 001 号实例）成为「能接单、能收钱、能结算、能扩展」的最小系统 | 用户能下单，艺术家能接单，钱能进来，收入能对账；三项可审计验证完成 |
| **治理层** | 创世公民就位；论坛、制度广场、艺术家直连区可运行 | 建「国」成功判定投票可发起；FCR 窗口可关闭 |
| **收入层** | 创世节点利益可验证兑现；联盟具备内生融资与滚动成长能力 | FCR 有流水；艺术家净收益优于单打独斗 |

## 二、有效路径总览（五步路径·大白话）

GAIA 不承诺一步到位，而是走一条**可验证、可追溯、可推进**的路径。五步用大白话解释如下：

| 步骤 | 术语 | 大白话 |
|------|------|--------|
| ① | 协议冻结 | **把规矩写死、定稿、打印成纸质。** 谁也不许随便改；要改得走正式流程。就像公司章程盖了章，白纸黑字摆在那儿。 |
| ② | 物理发生 | **东西真的做出来了、签了字、封了蜡。** 有一支真实存在的黄铜笔、一份火漆封存的协议书。拿得出、摸得着、拍得了照。不是嘴上说说。 |
| ③ | 外部确认 | **有「外人」认了。** 不只创始两人自说自话；至少有一个外部的人愿意签字、加入、背书。证明这不是二人转，而是有人真信。 |
| ④ | 平台上线 | **能接单、能收钱、能分账了。** 客户能下单，艺术家能接单，钱能进账、能按规矩分出去。从「在建」变成「在运营」。 |
| ⑤ | 多节点扩展 | **更多人加入、更多艺术家上架、更多客户下单。** 系统从几个人变成一群人，能自己滚动起来。 |

每一步都有现实抓手，每一步都可以被审计与复盘。详见下文 Phase A–E 的具象动作。

---

### 2.0 五层推进模型（技术对应）

```
Layer 0：意志层（Founder）
    ↓
Layer 1：文件层（协议冻结）     ← ①
    ↓
Layer 2：物理层（001 号套件）   ← ②
    ↓
Layer 3：记录层（影像/存证/论坛）← ③ 外部确认
    ↓
Layer 4：外部节点               ← ④⑤
```

**原则**：每一步最多推进 1 层；不允许跨层（如没文件就找人，没实物就拍片）。

### 2.1 创世期最优路线（对创世者与 CTO）

| 阶段 | 动作 | 产出 |
|------|------|------|
| **提前准备** | 建国路线图网站清晰可分享；核心协议（分润、FCR、决策权、过渡期）二人无分歧；可展示、可链接 | 条件就绪，可对外展示 |
| **二人完全一致** | 创世者与创世技术官就核心条款达成共识；无内耗；可统一话术 | 内部对齐 |
| **论坛优先** | CTO 先搭**论坛、制度广场**——建「国」路线图专区、艺术家×系统需求区；能说话、能讨论、能见路线图；**交易网站可后置** | 社区先于交易；见《公民参与与贡献记账》《创世贡献记录与确认机制》 |
| **大量招人** | 创世者招**艺术家 + 投资人**；CTO 招**IT 入伙人**；各自拉人，指向同一站点；广发英雄贴 | 节点扩张 |
| **交易系统** | 社区有活力、有人愿上架后再建；不阻塞招人与凝聚 | 分阶段实现 |

**分工**：创世者管艺术家与投资人；CTO 管 IT 与论坛/制度广场。**论坛、制度广场、艺术家直连区**为凝聚人心的核心基础设施，先于交易系统。

**配套**：[《节点角色说服与加入路径分析》](节点角色说服与加入路径分析（决策参考）.md)、[《建国务实进度看板：设计说明》](../02_协议与规范/建国务实进度看板：设计说明.md)、[《广发英雄贴：设计与操作说明》](../02_协议与规范/广发英雄贴：设计与操作说明.md)

## 三、时间节点安排（路线图）

### 锚点与相对时间

| 锚点 | 含义 |
|------|------|
| **T0 / Day 0** | 建「国」启动日；书面约定参与节点、权重、门槛；存入 Governance Log |
| **Day 90** | FCR 首期窗口参考届满；可发起建"国"成功投票；*小步快走路径*：可延后，以造血临界为实质触发 |
| **建"国"成功** | 投票通过则关闭 FCR 窗口；Batch 0 锁定；Batch 1+ 启动；*可选*：造血临界达成 + 投票 |
| **平台上线** | WhoAmI.Art 能接单、能收钱、能分账；首笔跨境款项到账；*小步路径*：可先人工记账接单 |

### Phase A｜协议冻结期（Day 1–15）

| 时间节点 | 动作 | 产出物 |
|----------|------|--------|
| Day 1–5 | 整理并冻结核心文档：创意载体规范、FCR 1.1 及附录、CTO 技术共识、三项不可谈判权力、分润与财务逻辑 | GAIA 法理内核 |
| Day 6–10 | 统一文档编号；转为 PDF + 本地只读；打印一套 | 可签署的纸质协议 |
| Day 11–15 | 最小数字存在（建国路线图单页/静态站）；只展示、不接单 | 可引用的存在证据 URL |

### Phase B｜物理发生期（Day 16–45）

| 时间节点 | 动作 | 产出物 |
|----------|------|--------|
| Day 16–25 | 001 物理套件准备：黄铜笔打样、棉浆纸、钢笔、火漆、档案盒 | 001 实验物理套件（未使用态） |
| Day 26–35 | 零号时刻：用 001 钢笔签署 FCR；火漆封存；编号 | 已签署的实体协议原件 |
| Day 36–45 | 最小影像记录（签字、火漆、档案盒）；不剪辑 | 原始影像素材（时间戳） |

### Phase C｜外部确认期（Day 46–90）

| 时间节点 | 动作 | 产出物 |
|----------|------|--------|
| Day 46–60 | 论坛/制度广场上线；第一封私邀信发出；创世技术官为首选外部节点 | 第一位外部节点的确认 or 沉默 |
| Day 61–75 | 若有同意：安排签署；首份 FCR 授予书 | 第一位 Founding Contributor |
| Day 76–90 | 贡献记录、任务认领、建「国」进度可见；Day 90 发起建「国」成功投票 | 建「国」成功判定；FCR 窗口关闭 or 延长 |

### Phase D｜平台上线期（建「国」成功后）

| 时间节点 | 动作 | 产出物 |
|----------|------|--------|
| Sprint 1 | 项目基础、用户系统、艺术家系统、模板系统、订单系统 | 可运行的 MVP 骨架 |
| Sprint 2 | 支付、资金安全、分配引擎；按《GAIA IT 供应链准入与 API 隔离标准》采购或自研 | 能收钱、能分账 |
| 首批上线 | 首批创世艺术家（前 10 名）；三项可审计验证（信任、治理、经济） | 可接单、可结算；首笔跨境到账 |

### Phase E｜收入与滚动成长期

| 时间节点 | 动作 | 检验标准 |
|----------|------|----------|
| 12 月内 | 创世节点利益可验证兑现；FCR 有流水；建国信用池启动（Batch 1+ 回补） | 联盟具备内生融资与滚动成长能力 |
| 18 月内 | 分配引擎连续运行；门类理事会、裁定者就位 | 治理层可独立运转 |
| 持续 | 物理载体可重复交付；基建池、建国回补机制运行 | 制度自动运转 |

## 四、关键路径图（Critical Path）

### 4.1 标准路径

```
T0 ──► Phase A (协议) ──► Phase B (物理) ──► Phase C (外部) ──► 建"国"成功 ──► Phase D (平台) ──► Phase E (收入)
       │                    │                    │
       │                    │                    └── 首份 FCR 授予
       │                    │                        建"国"成功投票（Day 90 参考）
       │                    └── 001 物理套件
       └── 核心文档冻结
```

### 4.2 小步快走路径（可选）

```
T0 ──► Phase A+B ──► 首单（人工记账） ──► 爬坡 ──► 造血临界 ──► 建"国"成功投票 ──► Phase E
       │                    │                │           │
       │                    │                │           └── FCR-3 完整；Batch 0 锁定
       │                    └── FCR-1 简化   └── FCR-2 扩大
       └── 协议 + 001
```

*详见《建国方案：小步快走与多期 FCR 设计（草案）》。*

**阻塞点**：
- Phase B 依赖物理载体供应链；须提前激活
- Phase C 依赖论坛上线、私邀响应；须话术到位
- 建「国」成功依赖 Day 0–7 权重约定、最低 5 人参与；*小步路径*：可延后至造血临界 + 投票
- Phase E 依赖 Phase D 上线；须分配引擎与支付打通；*小步路径*：可先人工记账

## 五、创世启动行动清单（Day 0–7 必做）

**启动时**（见《建国成功判定与窗口操作速查表》§一）：

1. 书面约定**参与节点名单**（创世公民、已签 FCR 者、forum 认领/附和≥1 次者）
2. 书面约定**权重分配**（创世者 30%–40%；其余按角色或 FCR 比例）
3. 约定**通过门槛**（如 2/3 或 75% 权重之和）
4. 整理并冻结核心文档（创意载体规范、FCR、CTO 共识、三项权力、分润逻辑）
5. 确认 001 实物采购清单
6. 选一个最小官网/建国路线图入口
7. 打印第一份 FCR；准备签署工具；预留「零号时刻」日期
8. 列出 3 个可能被私邀的人名

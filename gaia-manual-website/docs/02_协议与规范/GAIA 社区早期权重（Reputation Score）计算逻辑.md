# GAIA 社区早期权重（Reputation Score）计算逻辑

**用途**：将「发言、挑刺、拉人」等轻量参与转化为可量化、可展示的**参与度权重**；引导围观者→发言者→初代公民的转化；为 FCR 评估提供可操作输入；实现**社会资本自动化募集**  
**版本**：草案 v0.1  
**性质**：协议附件，与《创世贡献记录与确认机制（论坛版）》《FCR 附录》配套  
**设计意图**：将「无用的噪音」转化为「系统的算力」；注意力挖矿、零边际成本的社会资本置换高昂账面资本

---

## 一、核心定位

| 概念 | 含义 | 与 FCR 关系 |
|------|------|-------------|
| **Reputation Score（社区权重）** | 参与度、可见度、贡献密度的**可量化展示**；供本人与他人查看；引导行为、降低加入门槛 | **FCR 评估的参考输入之一**；参与度高≠自动获得 FCR 比例；有实质贡献方进入 FCR |
| **FCR 经济权利** | 从 20% 池结余按约定比例分配；90 日内实质性贡献；确权时锁定 | 社区权重高者，在 FCR 协商与评估时具有**参考优势**；二者不直接等同 |

**一句话**：社区权重让你「被看见」；FCR 让你「拿钱」。权重积累有助于获得 FCR，但 FCR 需实质性贡献 + 书面协议。

**与 C/G 分类的关系**：本逻辑为《公民参与与贡献记账》之操作化；C 类（共识贡献）对应发言、挑刺、提案、评议；G 类（增长贡献）对应拉人、带来订单/艺术家/渠道。详见[《公民参与与贡献记账》](公民参与与贡献记账（Public%20Participation%20%26%20Credits）.md)。

## 二、参与度 vs 权重（核心区分）

### 2.1 硬规则：发言本身不产生权重

> **发言本身不产生权重；被采纳、被引用、被验证的贡献才产生权重。**

| 概念 | 含义 | 产生方式 |
|------|------|----------|
| **参与度** | 发帖数、首参与日期、附和次数；可记录、可展示；让人「被看见」 | 任何有效发言、附和；计入展示，**不进入权重公式** |
| **权重（RP）** | 用于排序、高亮、可见度的量化值；决定发言在讨论中的显眼程度 | **仅来自**：被采纳建议、被验证批评、有效邀请、首参与奖励 |

**设计意图**：避免「发言就加分」导致灌水、权力崇拜；权重=低成本警察，而非统治工具。

### 2.2 产生权重的行为（示意）

以 **RP（Reputation Point）** 为计量单位；具体数值可经创世期共识微调。

| 贡献类型 | 权重单位（示意） | 说明 | 防滥用 |
|----------|------------------|------|--------|
| **建议被采纳** | 3 RP/次 | 创世者或确认人标记「已采纳」；对应协议或任务的实际改进 | 需书面或论坛记录 |
| **建设性批评被验证** | 1.5 RP/帖 | 指出逻辑漏洞、规则矛盾；经确认人标记为「有效批评」并验证 | 须有具体论点；单纯情绪宣泄不计入 |
| **发言被引用/附和达阈值** | 0.5–1 RP/次 | 他人引用、附和形成社会验证；具体阈值由创世期共识约定 | 自附和无效；防刷上限 |
| **有效邀请（拉人）** | 2 RP/人 | 直接邀请且被邀请人完成签约或实质贡献；须可追溯 | 同 IP、虚假注册不计入 |
| **首参与日期奖励** | 0.5 RP/日提前量 | Day 1 参与 vs Day 80 参与，前者获额外权重 | 与「风险对齐」逻辑一致 |

**不产生权重**：单纯发帖、单纯附和——仅计参与度（发帖数、附和次数），供展示与 FCR 参考，不进入 RP 公式。

### 2.3 围观者的路径

| 阶段 | 行为 | 参与度 | 权重（RP） | 转化逻辑 |
|------|------|--------|------------|----------|
| **纯围观** | 浏览文档、论坛，无发言 | 0 | 0 | 沉没成本心理：看得越多，越易迈出第一步 |
| **首次发言** | 发第一帖（哪怕质疑、挑刺） | 发帖数+1 | 0（除非被采纳/验证）| **挑刺即入门**；参与度可见→归属感 |
| **建议被采纳** | 创世者标记「已采纳」 | — | +3 RP | 实质性贡献→权重上升→发言高亮 |
| **拉人** | 邀请有效新成员 | — | +2 RP/人 | 亲友圈层的高信任转化 |

## 三、计算公式（简化版，待共识确定）

### 3.1 单用户 Reputation Score（权重公式）

```
Score = Σ(采纳建议×3 + 有效批评×1.5 + 被引用/附和达阈值×0.5~1 + 有效邀请×2) + 首参与奖励
```

**不进入公式**：单纯发帖、单纯附和——仅计参与度，不产生 RP。

**首参与奖励**（示意）：`0.5 × (90 - 首参与日序)`，即 Day 1 参与最多加 44.5，Day 90 参与为 0。

### 3.2 展示层级（Phase 1–3，与创世贡献机制对齐）

| Phase | 展示元素 | 实现难度 |
|-------|----------|----------|
| **Phase 1** | 发帖数、首参与日期、参与度（不含 RP） | 低；可手工或半自动 |
| **Phase 2** | 被附和次数、附和他人次数、**签约者身份标识**（头像/徽章可区分）、总 RP | 中；需确认人标记 |
| **Phase 3** | 建议被采纳数、有效邀请数、RP 排行、**权重参数公开可见**；权重高者发言默认高亮 | 中高；需与签约/入驻系统打通 |

**权重影响可见度，不影响发言权**：所有人都能发言；权重高→默认高亮；权重低→需被点赞/引用后上浮。引导而非统治。

**身份可见效应**：签约认可宪章者，头像/身份与权重在论坛内可查；人人可见谁在参与、谁有分量，极大促进自发维护者与建设者涌现。

## 四、与 FCR 评估的衔接

| 输入 | 用途 | 最终决定 |
|------|------|----------|
| **论坛记录** | 贡献自报、任务对应、产出物 | 人的判断 + 投票 |
| **Reputation Score** | 参与度、可见度、贡献密度的量化参考 | 供协商与加权时参照 |
| **邀请人数** | 直接邀请且成为创始贡献者的人数 | FCR 附录 §3.6：作为权重因素之一，不单独设池 |
| **任务难度、不可替代性** | FCR 附录 §2.2 量化原则 | 确权时比例约定的核心依据 |

**原则**：Reputation Score 不替代人的判断；高 RP 不自动等于高 FCR 比例；但 RP 为「谁在积极参与」提供可追溯、可审计的证据。

## 五、制度效应：为何能促进扩张

### 5.1 注意力挖矿

| 现象 | 制度含义 |
|------|----------|
| 围观者花时间阅读、思考 | **认知能量注入**；沉没成本促转化 |
| 挑刺者寻找漏洞 | **免费 QA 团队**；压力测试即价值；被验证后产生权重 |
| 发言者获得参与度 | **即时反馈**；「被看见」增强归属感；采纳后产生权重 |
| 拉人者获得 RP + 潜在分润 | **社交杠杆**；高信任圈层指数级拓扑 |

### 5.2 极低 CAC（获客成本）

- 传统平台：巨额买量成本
- GAIA：分配「未来治理权重」换取当下「社会算力」；**极低销售费用，极高品牌粘性**

### 5.3 全频谱协作网络

| 角色 | 贡献方式 | 系统价值 |
|------|----------|----------|
| 艺术家 | 创作、模板、首批供给 | 生产端 |
| 技术/运营 | 架构、获客、物流 | 基础设施 |
| **围观者** | 注意力、潜在转化 | 传播端 |
| **批评者** | 压力测试、规则改进 | 质量保证 |
| **拉人者** | 亲友圈层、高信任背书 | 扩张引擎 |
| **IT 在艺术家直连区有效回应** | 需求回应、采纳、解释 | 计入 IT 贡献权重；反向驱动 IT 盯住源头 |

## 六、防滥用与边界

| 风险 | 对策 |
|------|------|
| 灌水刷分 | 单日 RP 上限；附和权重低于发起；明显无意义帖不计入 |
| 虚假拉人 | 被邀请人须完成签约或实质贡献；同 IP、虚假注册不计入 |
| 批评滥用 | 「有效批评」须经确认人标记；须有具体论点 |
| 过度游戏化 | RP 为参考，非硬通货；FCR 经济权利仍由书面协议锁定 |
| **拉新层级** | 最多两层（推荐人/推荐人的推荐人）；不做无限层级返利 |

### 6.1 兑现原则（采纳/验证后才兑现）

**C 类贡献**：发言、挑刺计入参与度；**仅当建议被采纳或经评议人（Steward）标记为「可执行」**，方进入可兑现权重或获得额外声誉。避免「发言就有分」导致劣币驱逐良币。

**G 类贡献**：仅对真实订单、真实签约、真实合规节点奖励。

### 6.2 对外用词（合规护栏）

不说：空投、代币、挖矿、返佣。  
说：**贡献记账、声誉积分、公共信用、里程碑奖励**。

本机制不构成投资、证券或收益承诺。

## 七、生效与迭代

1. 权重单位、系数、上限经创世期共识确定后写入生效版；
2. Phase 1 可先行（发帖数、首参与日期）；Phase 2–3 随论坛与 IT 成熟逐步上线；
3. 与《创世贡献记录与确认机制》§3.1a、《FCR 附录》§1.6.4 保持一致。

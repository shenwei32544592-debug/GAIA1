建"国"债券发行与认购机制

用途：建"国"阶段按需募集资金；纯资本参与者路径；合规框架；创世者兜底保证募满
版本：1.0
性质：资金参与的设计说明，与《广发英雄贴》《建"国"务实进度看板》配套使用
合规口径：早期支持者认购；资金锁定支持建"国"；到期返还本金；建"国"者优先为非金融回报

一、核心逻辑

要素 设计
发债触发 每次有明确资金需求时发一期；不预先发大额，按需募资
认购资格 节点中的任何人可自由购买；配额与抢购见下
本金返回 创世资金有明确的返回机制；Escrow + 到期日 + 返还路径
创世者兜底 限时认购、抢购后仍有余的，由创世者认购；保证每期募满

二、发行流程

资金需求 → 发一期债 → 全节点等额配额 → 限时认购 → 未售出 → 抢购 → 仍有余 → 创世者兜底

2.1 配额（最简方法）

  • 全节点等额配额：本期总额 ÷ 发债时有效节点数 = 每人可认购量
  • 节点定义：创世公民、已签 FCR 者、forum 认领/附和≥1 次者；由创世期约定

2.2 限时认购

  • 设定截止日；节点在期限内认领本额
  • 先到先得本额

2.3 抢购

  • 逾期未售出部分进入抢购；开放给所有节点
  • 先到先得

2.4 兜底

  • 抢购后仍未售完的，由创世者认购
  • 保证每期募满

三、本金返回与提前退出

3.1 正常到期

  • 认购资金锁定于约定账户（Escrow 或专用账户）
  • 约定期限（如 90 日 / 180 日）
  • 到期按约定返还本金（原支付方式或转为平台信用）
  • 建"国"者优先等权益为非金融回报，单独约定

3.2 提前退出（可选）

锁定期内若允许提前退出，须在发债时约定：

项目 建议
是否允许 由每期发债条款约定；可允许、可禁止
条件 如：书面申请、提前 X 日通知
扣减 若允许，可约定轻微扣减（如 1–2%）以补偿流动性损失；或不扣减
目的 降低认购者流动性焦虑；具体以当期条款为准

四、对外合规口径

表述 用途
建"国"债券 / Founding Bonds / Founding Notes 工具名称
早期支持者认购 参与方式
资金锁定支持建"国" 用途
到期返还本金 资金去向
建"国"者优先、最惠国 非金融回报

避免：投资、回报、yield、股权、保证收益等表述。

五、与既有机制的关系

既有设计 债券对接
广发英雄贴 资金类需求可通过认购建"国"债券参与
论坛记录 认购凭证、金额、到期日可记录
看板 展示债券认购总额、认购人数、锁定期
确权 认购记录 = 资金贡献确权依据之一

# 建"国"务实进度看板：设计说明

**用途**：集中、动态展示联盟已聚集的力量；给已加入者信心，给观望者吸引力；为所有人（包括创世者）提供明确的信心来源  
**版本**：1.0  
**性质**：设计说明，与《创世贡献记录与确认机制（论坛版）》配套使用  
**原则**：正是源于这一切设计，方能实现前无古人的商业创新

---

## 一、目的与定位

### 1.1 核心功能

建"国"务实进度看板是**全息、动态、可见可感知**的信息集中展示与查询处，展示截止当前节点：

| 展示维度 | 示例 |
|----------|------|
| 已加入节点 | 有多少人、哪些角色、分布情况 |
| **各门类认领数与艺术家报到数** | 书法 X 人、非遗 Y 人、数字视觉 Z 人；**可见供给增长**，信号：平台有未来收入基础 |
| 已完成工作 | 任务完成数、关键产出、里程碑 |
| 募集资金 | 总额或档位、用途说明 |
| 任务认领 | 多少人认领了建"国"必要工作、哪些空缺 |
| 时间线与进度 | 阶段划分、距目标剩余距离 |

**门类可见性的设计意图**：节点看到「潜在艺术家这么多」→ 平台将来发财的大机会 → 促进其它想抓住机会的节点信心、加速真金投入。见《广发英雄贴》§2.6。

### 1.2 双重受众

| 受众 | 看板作用 |
|------|----------|
| **已加入者** | 看见同伴、看见贡献被记录、看见集体力量在增长；**信心来源** |
| **观望者** | 看见真实进展、看见谁在参与、看见计划与蓝图；**吸引力/说服力** |

### 1.3 信心来源原则

看板及整体制度设计，为**所有人（包括创世者）**提供明确的信心来源：

- 信心来自**可验证的设计与证据**，而非个人魅力或空泛承诺  
- 当节点在增、工作在进、蓝图可见时，创世者亦可从中获得支撑  
- **正是源于这一切设计，方能实现前无古人的商业创新**

### 1.4 创世期最优路线与 IT 优先级

| 原则 | 说明 |
|------|------|
| **凝聚人心优先于交易** | 论坛、广场（能说话、能讨论、能见路线图）先于交易网站；社区有活力再谈上架卖货 |
| **分工并行** | 创世者招艺术家 + 投资人；CTO 招 IT 入伙人 + 搭论坛/广场；二人一致后大量招人 |
| **提前准备** | 建"国"路线图网站清晰；二人核心条款无分歧；可展示、可链接后再扩张 |
| **看板与论坛** | 论坛负责记录与讨论；看板负责汇总与呈现；二者可分离亦可一体（如论坛自带汇总模块） |
| **建"国"路线图专区** | 论坛设专区承载网站全部内容；每条款、每文件可单独讨论；创世者重点回复答疑、即时改进；留存权威与公信力，透明化历史档案；见《创世贡献记录与确认机制》§3.1 |

## 二、双轨数据来源

### 2.1 节点自发布（约半数）

| 特点 | 说明 |
|------|------|
| **透明** | 已注册节点自行发布其认领任务、完成工作、加入时间等 |
| **可验证** | 他人可对照论坛、任务清单交叉核对；形成「我说、我证明」的证据链 |
| **去中心化** | 不依赖单一渠道汇总；节点对自己负责 |

### 2.2 创世者发布（约半数）

| 特点 | 说明 |
|------|------|
| **高效** | 仅创世者可见的全局数据（总资金、总人数、整体进度）由其一并披露 |
| **可信** | 创世者背书；避免重复统计与口径不一 |
| **全局视角** | 路线图、里程碑、阶段判断，需中心协调者整合 |

### 2.3 互补关系

- 节点自发布：**透明、可验证**  
- 创世者发布：**高效、可信**  
- 二者交叉印证，形成既有社会确认又有权威汇总的完整图景

## 三、可视叙事：王国城邦式呈现

### 3.1 呈现逻辑

创世者通过看板、三维图表或**王国城邦式图示**，使观者可感知：

| 感知目标 | 视觉手段 |
|----------|----------|
| **有计划** | 路线图、时间轴、阶段划分 |
| **有目的** | 目标拆解、使命说明、里程碑标签 |
| **有精细蓝图** | 架构图、分区图、建设蓝图式布局 |
| **进行中** | 进度条、完成度、准实时更新 |
| **胜利在望** | 完成比例、剩余距离、终点可见 |

### 3.2 王国城邦意象

| 意象 | 对应含义 |
|------|----------|
| **领土/版图** | 已覆盖领域、已加入节点分布 |
| **建筑/地基** | 协议、制度、基础设施完成度 |
| **城墙/堡垒** | 已建立能力、护城河 |
| **施工图** | 任务清单、阶段规划、分工 |
| **竣工倒计时** | 距首单、上线、制宪完成的时间与进度 |

### 3.3 形式服务于可感知性

图示形式服务于**让人感知**——有计划、有目的、有蓝图、在推进、胜利在望——而非技术炫耀。可选用：路线图、架构图、进度图、分区图、三维或平面示意，视实现成本与受众习惯而定。

## 四、公开与保密边界

| 建议公开 | 建议限制 |
|----------|----------|
| 节点数量、角色分布 | 具体人名（可经同意后展示） |
| 已完成任务数、认领数 | 个人 FCR 比例 |
| 募集总额或档位区间 | 单笔投资细节 |
| 时间线与里程碑进度 | 内部争议、未决事项 |

具体边界由创世期共识约定，并在看板页注明「披露规则」。

### 4.1 披露节奏

看板、池子余额、分配去向的更新节奏须写入约定，避免口头承诺：

| 项目 | 建议 |
|------|------|
| **看板更新** | 每周或双周；重大里程碑后 48 小时内 |
| **池子披露** | 每月或每季度；余额、本期分配去向、变更说明 |
| **决策留痕** | 重大决策（发债、延长窗口、池子用途调整）须有简要理由 + 日期，存入 Governance Log |

### 4.2 首单仪式

首单闭环达成时，看板触发**仪式化呈现**（置顶、公告、轻量庆祝）。将抽象数字转化为可感知的「建"国"发生」时刻；首批付费客户可纳入纪念身份（可选）。

## 五、与既有机制的关系

| 既有设计 | 看板可集成的数据源 |
|----------|----------------------|
| 论坛贡献记录 | 已完成工作、认领任务、参与人数 |
| 建"国"任务清单 | 任务总数、已认领数、空缺数 |
| 创世公民 | 已签署宪章者数量（可脱敏） |
| FCR 授予 | 创始贡献者数量（不含具体比例） |
| 募集/投资 | 资金进度（按披露规则） |

论坛负责记录，看板负责**汇总与呈现**；二者可分离，也可部分一体（如论坛自带汇总模块）。

## 六、改进建议（可选扩展）

以下为设计层面的可选增强，视资源与阶段采纳：

### 6.1 历史快照与版本追溯

- 每次重大更新留存**时间戳快照**（如 Day 30 视图、Day 60 视图）  
- 形成可回溯的历史记录；观者可感知「从无到有」的 momentum  
- 便于对比：建"国"前 vs 建"国"后

### 6.2 空洞位可视化

- 不仅展示「已完成」，也展示「待填充」：空缺任务、待建模块、缺口  
- 意象：城墙缺口、待建堡垒  
- 作用：明确「如何参与」「哪里需要人」；降低加入门槛，制造适度紧迫感

### 6.3 里程碑仪式与看板联动（见 §4.2）

- 当关键节点达成（如首 10 节点、首笔 $X 募集、**首单闭环**），看板触发仪式化呈现或公开公告  
- 首单达成时，首批付费客户可获纪念身份（可选）  
- 强化「胜利在望」的体验

### 6.4 对比维度

- 增加「行业惯例 vs 我们的设计」对比视图（在合规前提下）  
- 突显差异化：为何本联盟可做到前无古人的商业创新  
- 服务对外说服与对内锚定

### 6.5 轻量优先，迭代实现

- 创世期 IT 未必就绪时，可先以**人工维护的静态/半静态页面**或 Notion/Airtable 等低代码工具实现  
- 形式可简，原则不失：双轨数据、可视叙事、信心来源  
- 待系统成熟后，再升级为自动聚合、实时更新

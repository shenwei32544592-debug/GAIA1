广发英雄贴：设计与操作说明

用途：建"国"阶段广纳人才与资源；多多益善；先记录、后算账；信者自来
版本：1.0
性质：招募与权益承诺的设计说明，与《创世贡献记录与确认机制》《建"国"务实进度看板》配套使用
原则：建"国"成功前不做授勋论功；真正相信者自来，其余不勉强

一、核心逻辑

1.1 多多益善

建"国"过程吸引的人多多益善。来的人越多，集体希望感越强；不设人数上限。

1.2 广发英雄贴

列出联盟建"国"可能需要的全部资源、人才与需求类型，包括但不限于：

类型 示例
智力 战略、产品、制度设计、合规、法律
艺术 模板源头、审美标准、首批创作
技术 架构、实现、运维、安全
运营 获客、客服、供应链、物流
资金 投资、垫资、跨境结算；可经建"国"债券参与（见《建"国"债券发行与认购机制》）
社会资本 背书、引荐、行业准入

不定价：不事先为各类贡献设定具体价格；建"国"成功前无法精确估值。

1.3 两项承诺

承诺 内容
记录即依据 凡在约定论坛中被记录的贡献，将作为未来相对公平计算的依据
建"国"者优先 建"国"参与者天然享有：未来系统中的优先工作权、最惠国待遇，及可及利益的优先获得权

1.4 不提前论功

建"国"成功前不做授勋论功。精确算功、定价、分饼，此时既不可能也无必要。

1.5 信者自来

真正相信的人自然会来；不信者不强求、不赘述。英雄贴本身不解释过多；附「信者通道」链接（论坛、看板），好奇者自行深入。

1.6 招募开放与围观价值

当需要更多人时，任何节点均可招募新人；新人不论以何种角色进入，哪怕仅围观亦有价值——可能成为第一批客户、好奇者，或随时转化为主动贡献者。在建"国"需求推进过程中，参与者会自然聚拢，逐步形成节点分类（财务、法律、技术、运营等）。早期进入者（含围观）天然成为后期各类权益优先权的种子人选。

1.7 确权后的激励对齐

创始贡献者一经确权（FCR、Anchor 等），其最优策略转为做大系统——拉人、引入资源、促成增长。确权消除了「争 slice」的动机，代之以「做大 pie」；已确权者天然成为 recruiters，与创世目标无内在阻力。

1.8 邀请者分成（FCR 框架内）

被邀请者的部分价值在FCR 确权时作为分配权重因素之一纳入计算；不单独设池。须有可记录的邀请者；仅一代（A 邀 B，A 得；B 邀 C，仅 B 得 C 的份额，A 不因 C 获得）。以「人」为单位，不按订单细分，意在让大家都有邀请动力。

二、优化设计

2.1 需求分层

英雄贴中的需求按** urgency / 重要性**分层，便于观望者快速判断自己能参与哪一层：

层级 说明 示例
核心必需 缺则建"国"无法推进 协议起草、技术首版、首批资金
有益补充 加速或增强 合规咨询、设计、内容
长期储备 未来需要，可提前认领 多语种、多市场、衍生模块

2.2 最低参与门槛

明确最低动作即视为建"国"参与者：

  • 在论坛认领至少一项建"国"任务，或
  • 对他人贡献附和确认至少一次

完成其一，即享有「建"国"者优先」权益的资格；具体权益以协议约定为准。

2.3 与 FCR 的区分

入口 定位 权益形式
广发英雄贴 宽入口；先来先记；不设门槛 记录为依据、建"国"者优先、最惠国；具体比例后定
FCR 90 日 窄入口；需正式评估与授予 贡献值分配、永久署名；比例确权时书面约定；从 20% 公共运营池结余中分

二者并存：英雄贴吸纳广泛参与；FCR 覆盖需严格确权的核心贡献者。

2.4 创世后加入者不参与 FCR 补偿

创世成功后的任何新加入者（IT 人才、设计师、运营等）均不参与 FCR 及类似补偿机制。他们仅能：

方式 说明
工资激励 作为雇员/承包商，按劳务获取报酬
公民权创收 以艺术家身份获 65% 分润，或以平台参与者身份获得相应收益

此边界保证建"国"贡献者权益的稀缺性与不可复制性。详见《FCR 附录：90日窗口与价值量化说明》§4.2。

2.5 权益边界与既有条款衔接

「建"国"者优先」「最惠国待遇」与既有《衍生业务优先权与最惠国条款》衔接；英雄贴不做重复发明，仅明确：建"国"期参与者的认领与附和记录，构成享有该等权益的资格依据之一。具体边界以正式协议为准。

2.6 门类无痛认领与开通激励

无痛认领：艺术家可在签约前明确认领一个门类（如书法、非遗、数字视觉）；操作尽量简化（选门类、留联系方式）；认领≠签约，仅为占位与意向表达。降低心理门槛，先试水温再决定深度参与。

门类开通激励：若某人认领并开通一新门类（该门类尚无代表或未达开通条件），且引入 ≥3 名艺术家报到（在论坛对应门类下发言/认领），则享有未来好处：如该门类首位开通者可获 FCR 加权、或门类理事会代表优先资格；具体由创世期共识约定。

设计意图：各门类认领数、艺术家报到数在看板可见；节点看到「潜在艺术家这么多」→ 平台未来有供给基础 → 信心提升 → 促进其它节点加速真金投入。

三、操作要点

3.1 英雄贴发布

  • 创世者或指定人整理需求清单，按 2.1 分层
  • 发布至论坛或看板；可同步作为看板「空洞位可视化」的数据源
  • 清单可随进展增补；保持透明

3.2 认领与附和

  • 参与者于论坛认领任务，或对他人贡献附和
  • 记录自动进入《创世贡献记录与确认机制》的流水账
  • 与看板联动：认领数、附和数可展示于建"国"务实进度看板

3.3 信者通道

  • 英雄贴正文简洁；附链接：论坛、看板、任务清单
  • 不赘述制度细节；愿深入者自行点击

四、可行性概览

维度 评估
技术 论坛 + 清单页即可；可用 Notion/Markdown + Discord/飞书/开源论坛
效率 一次列出，长期有效；创世者维护成本低
资源 创世者整理清单 1–2 天；无额外开发
风险 需求过泛可稀释焦点→用分层控制；承诺过宽有法律风险→以协议为准、加限定语
优先级 可与建"国"任务清单、论坛贡献记录同步启动

五、与主协议的关系

  • 本说明不修改 FCR、创意载体规范、最惠国条款等主协议
  • 英雄贴为招募与记录层设计；权益兑现以正式协议为准
  • 若冲突，以主协议为准

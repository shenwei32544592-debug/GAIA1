论坛身份体系与贡献记账规则（MVP）

用途：论坛身份体系最小可行版本；入籍路径与身份层级；贡献记账一页纸；供 IT 实现与运营参考
版本：MVP v0.1
性质：操作指引，与《公民参与与贡献记账》《GAIA 社区早期权重计算逻辑》配套
设计意图：身份可见→秩序自发；克制设计→引导而非统治；发言不产生权重→避免灌水与权力崇拜

一、身份层级（最小可行）

身份 标识 权限 权重展示
游客 无头像/默认 可浏览制度广场；不可发言 无
注册者 自选头像 可发言、附和；参与度可见 发帖数、首参与日期
宪章签署人 签约后头像/徽章可区分 同上；身份明确标识 参与度 + RP（若产生）
创世贡献者 专属徽章 同上；FCR 相关权益另见协议 参与度 + RP + FCR 比例

入籍路径：围观→注册→阅读→签署宪章→身份变化（头像/徽章变更）→权重积累

二、贡献记账规则（一页纸）

2.1 什么会被记录

行为 参与度 权重（RP）
发帖 ✓ 发帖数+1 ✗ 不产生
附和 ✓ 附和次数+1 ✗ 不产生
建议被采纳 ✓ ✓ +3 RP
批评被验证 ✓ ✓ +1.5 RP
发言被引用/附和达阈值 ✓ ✓ +0.5~1 RP
有效邀请 ✓ ✓ +2 RP/人
首参与日期 ✓ ✓ 首参与奖励

2.2 什么不会

行为 处理
灌水、重复、无意义帖 不计入
单纯情绪宣泄 不计入
虚假拉人、同 IP 刷量 不计入
自附和 无效

2.3 硬规则

发言本身不产生权重；被采纳、被引用、被验证的贡献才产生权重。

三、权重与可见度

原则 内容
权重影响可见度，不影响发言权 所有人都能发言
权重高 发言默认高亮、易被看见
权重低 发言需被点赞/引用后上浮
引导，非统治 不是权力崇拜；是秩序显形

四、入籍路径文案（示意）

步骤 动作 文案/提示
1 围观 「欢迎浏览；制度广场开放阅读」
2 注册 「注册后可发言、参与讨论」
3 阅读 「建议先阅读宪章与理由书」
4 签署 「签署认可宪章，成为创世公民」
5 身份变化 「您的身份已更新；头像/徽章已激活」
6 贡献 「参与讨论、提出建议；被采纳即产生权重」

五、与主协议关系

本 MVP 为《公民参与与贡献记账》之操作化；若与主协议冲突，以主协议为准。

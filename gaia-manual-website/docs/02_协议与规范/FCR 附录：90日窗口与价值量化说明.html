FCR 附录：90日窗口与价值量化说明

用途：补充 FCR 1.1 的设计理由与操作指引；供创始人、制宪会议及潜在创始贡献者理解「为何90日」「如何量化」「如何公平化」
版本：1.0
性质：解释性附录，与 FCR 1.1 一并理解

〇、用语界定（FCR 1.1 配套）

用语 含义
建"国" 隐喻用法；「国」指艺术家联盟秩序（艺术家的理想国），非政治实体；WhoAmI.Art 及经其授权、承接贡献记录与确权的载体所承载的创世期建设活动；不锁定单一域名或站点；悬停可浮出释义
本平台 WhoAmI.Art 及其授权平台与产品；贡献记录、确权、分配可在其后台、论坛、看板或授权网站完成

一、为何是 90 日？

1.1 风险对齐（Risk Alignment）

创始贡献者身份的核心价值在于：在系统最不确定时承担了不可逆的投入。

时间段 不确定性 投入性质
Day 0–90 最高：无订单、无验证、无外部背书 智力、时间、声誉、机会成本；不可逆
Day 90+ 降低：平台上线、首批艺术家、首单闭环可期 后续参与方有可验证案例可参考；风险可评估

90 日窗口确保：只有承担了最高不确定性的贡献者，才享有创始贡献者权利。晚于 90 日加入者，可获取 Batch 1+ 的标准化权益（如 65% 分润、Kill Switch），但不再具备创始贡献者身份。

1.2 稀缺性与防通胀

若无时间边界，「创始贡献者」将随业务成长而膨胀，导致：

  • 权益池被持续稀释；
  • 「创始」一词丧失意义；
  • 早期贡献者的回报被摊薄。

90 日为基准窗口；稀缺性由「建"国"成功」与时间双重边界共同建立。见下 §1.6。

1.3 业务周期与可验证性

90 日约等于一个季度，是常见的业务验证周期：

  • 足以完成：协议冻结、宪章签署、首批技术交付、平台上线准备；
  • 不足以：让「观望者」等到业务验证后再以低风险加入并 claiming 创始身份。

《有效路径与时间节点安排》中，Phase D（平台上线）约在 Day 90 附近；FCR 窗口与「创世期→运营期」转折点对齐。窗口关闭时，外部艺术家开始批量加入，此时「创始」与「后续」的界线已清晰。

1.4 法理与惯例参照

90 日常见于：

  • 劳动法试用期（多法域）；
  • 修正案披露期（《三项不可谈判权力声明》等）；
  • 签字权承继程序的紧急期限（《签字权与法人地位承继规则》）。

选用 90 日，便于与既有协议用语一致，降低解释成本。

1.5 博弈论视角

窗口长度 优点 缺点
过短（如 30 日） 稀缺性极强 难以完成实质性贡献；可能遗漏关键节点
90 日 足以完成制宪+首版交付；与业务节点对齐 平衡点
过长（如 180 日+） 容纳更多参与者 稀释稀缺性；晚期加入者风险已显著降低

1.6 90 日的机动边界：建"国"成功与适当延长

90 日非绝对刚性；以「建"国"成功」为实质锚点。建"国"成功与否，原则上由有权重比例的节点投票结果为准；创世者裁量权为补充与兜底。

小步快走路径（可选）：建国锁定期可延后至造血临界。以「造血临界达成 + 投票通过」为建国成功；90 日为参考窗口，非刚性。详见《建国方案：小步快走与多期 FCR 设计（草案）》。造血临界示意：平台 500 单/天 或 月收入 37.5 万；时间兜底：启动后不晚于 18 月。

1.6.1 投票判定机制

项目 约定
判定依据 有权重比例的节点投票；赞成达到权重之和的**≥ 某比例**（如 2/3 或 75%）即为通过
最低参与人数 创世公民 签署人**≥ 5 人**（含创世者、创世技术官）时，投票具效力；建议目标 8–10 人；投资者、家人、艺术家为利益相关方
参与节点 创世公民 签署人、已签 FCR 授予书者、 forum 认领或附和≥1 次的建"国"参与者；具体由创世期共识约定
权重设计 创世者因信息最完整，享有较高权重（建议 30%–40%）；其余按角色或 FCR 比例分配；具体权重须在创世期启动时书面约定，避免事后争议
兜底 参与不足（如有效参与者 < 5 人）或无法形成有效投票时，以创世者判断为准
创世者异议 若创世者认为投票结果明显偏离事实，可说明理由并要求重投或暂缓执行；重投以一次为限
公民修正征集 首届治理就位后 12 个月内，须完成第一次「公民修正提案征集」；广泛民意通过持续修正与参与实现

1.6.2 情形与处理

情形 窗口处理
建"国"提前成功 创世者或节点提议发起投票；投票通过则提前关闭窗口，不再新增创始贡献者
90 日届满 届时发起「建"国"是否成功」投票；通过则关闭；未通过则进入「是否延长」程序
适当延长 若投票判定未成功，可再投票决定是否延长、延长多久（如 30 日）、延长条件；或由创世者提议，投票确认
创世者裁量 制度无法穷尽的情形（如紧急、僵局、程序无法启动），以创世者判断为准

1.6.3 快速操作指引（建"国"启动时可用）

  1. 启动时：书面约定参与节点名单、权重分配、通过门槛，存入 Governance Log
  2. Day 90 或提议时：创世者或≥1 名节点发起投票；在论坛或约定渠道公告议题与截止时间
  3. 投票：各节点按权重投票（赞成/反对/弃权）；汇总权重，判断是否达到门槛
  4. 结果：公示结果；通过则关闭窗口或延长；未通过则维持现状或重投
  5. 兜底：参与不足或程序受阻时，创世者裁量并书面说明理由

速查：建"国"启动时可使用《建"国"成功判定与窗口操作速查表》快速查阅流程与步骤。

1.6.4 多期 FCR 操作指引

采用小步快走路径时，可配置多期 FCR 结构：FCR-1（首单期，简化比例）→ FCR-2（爬坡期，扩大池）→ FCR-3（造血临界，完整结构、Batch 0 锁定）。各期比例、触发条件、剩余空间须书面约定，避免稀释争议。详见《建国方案：小步快走与多期 FCR 设计（草案）》。

1.6.5 论坛参与度与 FCR 评估

论坛发言即贡献可见（见《创世贡献记录与确认机制》§3.1a）：发帖数、首参与日期、附和、被采纳建议、建设性批评、有效邀请等形成账号参与度。参与度可量化为 Reputation Score（社区权重）；计算逻辑见《GAIA 社区早期权重计算逻辑》。参与度为 FCR 评估的参考输入之一，与任务完成、产出物、邀请人数等一并纳入；人的判断与投票为最终依据。围观者发言同样计入参与度，有助于潜在公民留存与转化；参与度≠FCR 比例。

1.7 创世者提名权与裁量权

背景：创世过程中，部分贡献难以简单评估——如特殊人才、稀缺资源、基于对创世者信任而先予投入者。规则难以穷举。

设计：创世者享有提名权——对难以简单评估的贡献，可提名其为创始贡献者，并在合理范围内约定比例。此裁量权服务于：

  • 机动性：在条规则下容纳「相信我的权利而贡献」的特殊人才与资源；
  • 公平的实质：形式公平（论坛记录、加权投票）与实质公平（创世者识别隐形贡献）互补；
  • 边界：提名须经书面《FCR 授予书》记载；不得滥用；受 FCR 1.1 第四条撤销条款约束。

二、价值如何被量化？

2.1 本协议约定：比例在 FCR 确权时确定

FCR 1.1 第二条 2.1 约定：

  • 分配比例于 FCR 确权时确定（基于可量化的贡献值）；
  • 一经接受即固定不变；
  • 不接受追溯性比例稀释。

与艺术家版权/版税的区别：FCR 不涉及版权、版税。艺术家拿 65%是基于其创作与 IP（详见《递归反哺与源头确权细则》）；FCR 是建"国"期贡献者基于可量化贡献值从 20% 池结余中按约定比例分配（详见《分润与财务逻辑补充》）。二者池子不同、逻辑不同。

池子来源：20% 池由全平台订单收入的 Net Pool 注入；FCR 持有者按约定比例从池结余中获取。无订单则池空，无此部分收益。

大白话：确权时把你的贡献折成可量化的值，约定从池子拿多少比例。比例写进协议就定死；后来的人不会把你的那份挤小。

举例：李四做了技术架构，确权时约定从 20% 池结余拿 3%。一年后平台来了 20 个新贡献者，李四还是 3%。

2.2 贡献类型与权重参考

贡献类型（FCR 1.1 第一条 1.1）：

  • 智力：战略、产品设计、制度设计、规则起草、合规、法律；
  • 艺术：模板库源头、首批可商用创作、审美标准贡献；
  • 基础设施：技术架构、实现、运维、合规与财务体系；
  • 资金：投资、垫资、建"国"债券认购、跨境结算支持；
  • 运营：获客、客服、供应链、物流；
  • 社会资本：背书、引荐、行业准入、资源对接。

量化原则（非硬公式，为 Founder 与潜在贡献者协商参照）：

维度 考量
不可替代性 该贡献若缺失，系统能否成立或上线？
时间投入 在 90 日窗口内的实际投入密度
风险承担 是否放弃了其他确定性收入或机会
模块边界 贡献是否对应「可独立识别的模块或成果」；可量化贡献值在确权时折成分配比例

2.3 与 创世公民 / Anchor 的区分

概念 来源 量化方式
FCR（贡献值分配） FCR 1.1 确权时基于可量化贡献值约定分配比例；从 20% 池结余按比例获取
创世公民（5% Anchor Floor） 递归反哺、分润逻辑 签自然人宪章；享有 5% 基准桶；Time Decay 18 月→50%；Cap 200%；订单量级触发衰减

FCR 与 创世公民 可重叠（如创世技术官既获 FCR 又属 创世公民），但二者来源不同、计算逻辑不同。FCR 为「贡献值分配」；创世公民 为「治理基准池的创世公民分配」。

2.4 操作要求：书面化

《Founder-沈炎君 协作与治理要点》：用 FCR 协议明确比例，不以口头对半分为准。

量化公平化的底线：所有比例、贡献描述、模块边界，须写入《FCR 授予书》或等效书面协议，经双方签署。口头承诺不作数。

2.5 比例确定机制：系统参考 + 协商 + 同行评议

环节 说明
① 系统参考区间 平台公布按贡献类型的建议比例区间（如：模板/艺术类 0.2%–1%、技术/架构类 2%–5%、制度设计类 1%–3%），供协商参照；区间可随共识调整
② 本人与创世者协商 贡献者与创世者（或授权代表）在参考区间内，或附理由在区间外，达成一致
③ 书面锁定 比例写入《FCR 授予书》，双方签署即生效
④ 同行评议（可选） 当比例超参考区间上限、或任一方提请时，可请求门类理事会（同门类）或 裁定者 出具公平性意见；意见供参考，不替代双方已达成的协议；创世者保留最终裁量

大白话：系统先给出「一般人拿多少」的参考区间；你和创世者谈，谈妥就签。如果比例特别高、有人觉得不公平，可以请同行（门类理事会或裁定者）看看、给个意见，但最终以你和创世者签的协议为准。

与《分配方案：公式化定档与供应商定价》的关系：采用多期 FCR 时，贡献定档以该方案中的公式化区间为准，消除双边谈判压价；比例确定仍以协商共识为主线， Founder 裁决为边际兜底；裁决须保障异议通道。详见该方案 §第一条、§第三条。

三、如何被公平化？

3.1 透明化

  • 贡献类型、权重考量、模块边界，在协商时对贡献者透明；
  • 创始人说明比例依据，不隐瞒判断逻辑；
  • 争议时，书面协议为唯一依据。

3.2 不可稀释

  • 一经接受，固定不变；
  • 后续新增创始贡献者，不稀释既有贡献者的比例；
  • 比例在既有贡献者之间如何切分，由创始人根据入驻时的约定分配；新加入者从「剩余空间」或独立池中获取，不触碰既有人。

3.3 可审计

  • 分配指令由 RuleID 生成，会计与结算须服从 RuleID；
  • 任何人不得 override 已生效的分配结果；
  • 流水可追溯，全平台可审计。

3.4 权利撤销有边界

FCR 1.1 第四条规定：仅在规避协议、引入未授权生成式系统、损害载体完整性时，可撤销。
撤销权由创始人专属行使，不可滥用；贡献者有权知悉撤销理由及程序。

3.5 「够不够」的锚点

《Founder-沈炎君 协作与治理要点》：收入达生活保障、家庭压力缓解、贡献者感到被公平对待。

公平不仅来自公式，也来自心理预期与制度承诺的匹配。协议透明 + 收入优先 + 明确认可，比模糊承诺更利于长期合作。

3.6 邀请因素

在 FCR 确权时，将「直接邀请人数」（或其中成为创始贡献者的人数）作为分配权重因素之一纳入计算。不单独设池；仅作为确权公式的一个输入项。

四、与主协议的关系

本附录不修改 FCR 1.1 的实质性条款；仅对 90 日窗口与价值量化的设计理由与操作指引作补充说明。
若附录与主协议冲突，以 FCR 1.1 为准。

四之二、创世后加入者不参与补偿机制

4.2.1 资金来源

FCR（创始贡献者权利）权益来自20% 公共运营池的结余部分。该池先覆盖运营成本，结余按《FCR 授予书》约定比例分配给创始贡献者。详见《分润与财务逻辑补充》§五。

4.2.2 参与边界

创世成功后的任何新加入者——无论 IT 人才、设计师、运营或其他——均不参与此 FCR 分配机制。

对象 补偿方式
创始贡献者（90 日内） FCR 贡献值分配，从 20% 池结余中分
创世后加入者 仅能通过（1）工资激励（劳务报酬）或（2）公民权创收（如以艺术家身份获得 65% 分润、以平台参与者身份获得相应收益）

4.2.3 含义

  • 建"国"阶段的贡献者享有一次性、不可复制的权益锁定；
  • 建"国"成功后再加入的人才，按市场规则参与：要么受雇拿工资，要么通过公民身份（艺术家、服务提供者等）在平台内创收。
  • 此边界避免「后来者搭便车」；激励真正承担建"国"风险者。

4.2.4 建国回补与建国信用池

后进艺术家（Batch 1+）享受「确定性红利」——系统已跑通、品牌已建立。联盟设立建国回补机制：后进艺术家在加入后短期内（如 90 天或 30 单），以其 Artist Pool 的 1%–3% 进入建国信用池，按 FCR 权重分配予创世公民。回补为限期机制，到期自动停止。详见《建国回补与确定性溢价机制（草案）》。

四之三、FCR 期限、池子与治理层

4.3.1 期限与回报（参考派发新股，原始股权不动）

建"国"成功后 FCR 分配表锁定（每人原始股数固定）。参考派发新股机制：

  • 派发新股：当平台成长、需引入新人才时，为新加入者派发新股；既有者的原始股权数量不变
  • 计算时自动反映：分配收益时，总股数 = 原有 + 新股；每人实得 = （其股数 ÷ 总股数）× 池子。总股数变化自动带来每人比例变化，无需修改任何人的记录
  • 或什么都不变动：既有人记录完全不 touch；只增新人的股；计算层自然处理
  • 心理：人们一直看着自己有多少原始股权没有变动，心也安；比例变化是无形的，不需面对「被削减」的观感

4.3.2 池子多用途与透明度

FCR 分配及未来人才激励所涉资金，来自多用途运营池。该池可优先用于：外包激励、新人激励、FCR 分配及其他运营需要；用途与优先级由治理约定，可调整。本池不提供专款或最低保障承诺。池子余额、分配去向应定期披露。

4.3.3 治理层与池子的利益对齐

建"国"功臣中必有进入治理层的节点。池子用途由治理层决定。FCR 持有者参与治理时，会自然平衡：人才引进多、激励得当→总盘子大→FCR 回报更好；不必要的奖励=稀释自身收益，故不会主动滥发。

4.3.4 治理层与 FCR 脱钩

治理层以平台发展与公共善为导向，不将进入资格与 FCR 持有绑定。FCR 持有者已享有经济回报；若再因 FCR 自动获得治理权，可能成为平台变革的阻力。治理层由 创世公民、门类理事会、艺术家代表等多元角色组成；进入资格基于角色与能力，而非 FCR。借鉴美国宪政：经济权益与治理权分离；治理服务于共同体，非单一利益群体。

五、IT 未就绪时的过渡机制

90 日内，分配引擎、RuleID 可能尚未上线，贡献难以被 IT 自动记录。以下机制确保贡献者被看见、未来可期：

5.1 书面协议优先

  • 《FCR 授予书》在入驻时签订，写清：贡献模块、对应比例、触发条件
  • 即使暂无「全球使用情况」数据，协议已锁定：一旦有收入，按约定比例分配
  • 书面协议 = 法律承诺，优先于「等系统记录后再谈」

5.2 论坛作为贡献流水账

  • 采用公共论坛作为 90 日内贡献记录载体；详见《创世贡献记录与确认机制（论坛版）》
  • 创世者发布建"国"必要任务清单；贡献者自报工作与讨论；他人附和确认；形成可追溯、可确权的记录
  • 论坛记录为后续《FCR 授予书》比例约定的依据之一

5.3 盘点节奏

  • 不必固定周期；可按 Sprint 完成、协议冻结等关键里程碑做阶段性贡献小结
  • 创世者或指定确认人，对每位贡献者做简要盘点：「我们如何识别并记载你的贡献」
  • 目的：让贡献者感到被看见，而非仅依赖系统自动抓取

5.4 署名权先行

  • FCR 的署名权不依赖 IT，可立即体现
  • 在文档、协议、对外材料中，明确列出创始贡献者名单与角色
  • 允许使用「创始贡献者」专属标识；署名即时而可核验

5.5 仪式感

  • 零号时刻：用 001 钢笔签署 FCR、火漆封存、编号、影像记录（见《有效路径》）
  • 让承诺有形可触；产生可留存、可展示的物理证据；传达「会兑现」的信号

# 第一层：商业逻辑

## 1.1 核心命题

| 问题 | 回答 |
|------|------|
| 我们做什么？ | 用签名作为入口产品，连接中国艺术家与全球市场；平台提供履约、分配、透明机制 |
| 为什么能成？ | 红人出海是趋势；艺术家已有流量缺变现；平台提供美国落地 + 分钱机制 |
| 与别人的差异？ | 不是工具，是「身份表达 + 艺术 + 分配机制」；分钱机制是我们的护城河 |

## 1.2 趋势：红人出海

| 维度 | 现状 |
|------|------|
| 中国艺术家 | 飘漆、软包、书法等品类，国内已做大规模，流量大 |
| 痛点 | 变现主要靠广告，执行与落地能力有限 |
| 需求 | 出海、产品化、可持续收入 |
| 我们 | 签名切入 + 美国履约 + 分钱机制 |

## 1.3 产品利润逻辑（全球通用）

| 逻辑链 | 内容 |
|--------|------|
| 艺术家 | 创作 + 自媒体内容（平台绑定） |
| 平台 | 协助推广 → 自然流量 |
| 流量 | 连接目标市场，平台监控转化 |
| 履约 | 美国仓储、发货、送达 |
| 收益 | 订单收入 → 按规则分配 |

## 1.4 唯一变量：如何分钱

| 要点 | 内容 |
|------|------|
| 核心差异 | 趋势、产品逻辑别人也能做；**分钱机制**是我们的核心 |
| 机制要求 | 透明、可追溯、可计算、写入协议 |
| 参与者认知 | 分润权来自系统，不来自某一公司；签约对应公司不重要，机制重要 |

## 1.5 品牌：WhoAmI.Art

| 要素 | 内容 |
|------|------|
| 域名 | 已注册 |
| 语义 | 「我是谁」的艺术表达；普世追问，易记易传 |
| 后缀 | .Art 匹配艺术、身份美学 |

# 第二层：分配机制

## 2.1 基本原则

| 原则 | 内容 |
|------|------|
| 无人持股 | 不给股权，给分润权（合同约定） |
| 按贡献分润 | 创意、技术、劳动、投资均有回报路径 |
| 早期优先 | 首批加入者回报高于后期加入者 |
| 透明可追溯 | 每笔分配有来源、有规则、可查 |

## 2.2 分润权来源

| 来源 | 分润依据 |
|------|----------|
| 贡献 | 创意、技术、劳动、运营 → 贡献评估 / 贡献凭证 |
| 投资 | 资金、资源 → 投资协议、回报条款 |
| 参与 | 艺术家、合作方 → 分成协议 |

## 2.3 早期优先机制（首批参与者必读）

| 机制 | 说明 |
|------|------|
| **FCR（创始贡献者权利）** | 平台早期营收阶段，创始贡献者享有优先分润权 |
| **风险补偿基金** | 前期出资人在后期从风险基金中获得补偿 |
| **贡献时间加权** | 同等贡献，参与时间越早，权重越高 |
| **批次档位** | 第 1 批 > 第 2 批 > 第 3 批… |

**首批参与者**：艺术家、投资人、IT、供应商、履约方——你们的早期投入，会在机制中被明确承认并优先回报。

## 2.4 协商机制

| 机制 | 内容 |
|------|------|
| 逆向投票 | 每人说出真实需求 → 匹配机制找交集 → 形成共识 |
| 分配规则 | 贡献参数、回报参数均有标准；系统可给出参考 |
| 争议处理 | 人工复核 + 记录；重大争议经起草委员会或仲裁 |

## 2.5 宪政与签约

| 要点 | 内容 |
|------|------|
| 人人签约 | 参与者与公司签署协议，承诺遵守宪政与分配规则 |
| 法律效力 | 协议具法律约束力；违约可诉 |
| 公开可见 | 谁签了、签什么，在讨论广场可见 |
| 入口暂在 Founder 名下 | 运营实体暂由 Founder 代表；分润权来自系统，不依附某一法人 |

# 第三层：IT 机制

## 3.1 系统架构概览

| 系统 | 功能 | 阶段 |
|------|------|------|
| 平台官网 | WhoAmI.Art 展示、导流 | Phase 1 |
| 认证系统 | 节点登录、身份验证 | Phase 1 |
| 节点/贡献系统 | 登记节点、记录贡献 | Phase 1 |
| 分配引擎 | 收入 → 规则 → 分配流水 | Phase 1 |
| 签约/协议系统 | 签署宪政承诺、存证 | Phase 1 |
| 讨论广场 | 讨论、投票、草案展示、贡献记录 | Phase 1 |
| Demo 模拟 | 订单 → 计算 → 分配演示 | Phase 1 |
| 支付 | 收款、分账 | Phase 2 |
| 履约对接 | 订单 → 仓储/物流 | Phase 2 |

## 3.2 讨论广场（证据 + 信用）

| 作用 | 内容 |
|------|------|
| 吸引任何人 | 新来者、第三方可进入 |
| 可回溯历史 | 讨论、决策、签约可查 |
| 证据系统 | 记录讨论内容、时间、参与人 |
| 信用系统 | 历史可见、可验证，形成声誉 |
| 贡献记录 | 发言次数、内容、长度、主动反馈程度 → 计入贡献值 |

**平台选择**：用成熟产品（如 Circle、Guild、Discourse），快速上线，不做从零自建。

## 3.3 机制与系统映射

| 机制 | 系统表现 |
|------|----------|
| 协商机制 | 节点提交需求 → 系统记录 → 匹配/回流输出 |
| 分配机制 | 收入入账 → 算法计算 → 分配记录 → 可追溯 |
| 追溯机制 | 每笔分配有 source_id、rule_id、可查 |
| 签约机制 | 节点签约 → agreements 表存证 |
| 贡献记录 | 讨论参与、订单贡献、投资等 → contributions 表 |

## 3.4 人员与软件依赖

| 角色 | 最少 | 依赖软件 |
|------|------|----------|
| Founder | 1 | 全部（总览、决策） |
| IT 工程师 | 2–3 | 全部系统 |
| 艺术家 | 2–3 起步 | 认证、贡献、分配查询、讨论广场 |
| 投资人 | 1–2 | 讨论广场、Demo、协议 |
| 美国履约方 | 1 家 | 履约对接、订单系统 |
| 供应商 | 1–2 家 | 订单、结算 |
| 法务/财务 | 可外包 | 协议、对账 |

# 第四层：推进步骤

## 4.1 总览

| 阶段 | 周期 | 关键动作 |
|------|------|----------|
| Step 0 | 1–2 周 | 草案定稿、逻辑图、讨论广场选型 |
| Step 1 | 2–3 周 | 节点表、签约流程、起草委员会首批召集 |
| Step 2 | 2–3 周 | 贡献记录、分配规则初版 |
| Step 3 | 3–4 周 | 分配引擎、流水、Demo |
| Step 4 | 1 周 | 节点端查询「我的贡献/分配」 |
| Step 5 | 持续 | 首单、履约、迭代 |

**合计**：约 8–12 周可完成最小闭环。

## 4.2 100 万人民币分配建议

| 用途 | 占比 | 说明 |
|------|------|------|
| IT 基础设施 | 25–30% | 平台、履约、分配算法 |
| 美国履约首期 | 20–25% | 仓储、物流、首批库存 |
| 首批艺术家签约与内容 | 15–20% | 签约费、素材、打样 |
| 品牌与获客 | 15–20% | 官网、TikTok 冷启动 |
| 法务与合规 | 10–15% | 协议、分配规则法律设计 |
| 预留 | 10% | 不可预见 |

## 4.3 双实体结构

| 实体 | 所在地 | 职能 |
|------|--------|------|
| 美国法人（如 LLC） | 美国 | 签约、履约、平台运营 |
| 中国法人（如有限公司） | 中国 | 资金、供应链、艺术家对接 |

两实体通过协议连接，均服从同一宪政与分配机制。**核心资产在平台层，不归属于任一法人；参与者分润权来自系统。**

## 4.4 法律与信任

| 维度 | 内容 |
|------|------|
| 法律 | 适用美国法、仲裁条款；协议对各地参与者有约束力 |
| 信任 | 透明记录、公开签约、可追溯分配 |
| 关系 | 法律兜底，信任支撑日常 |

# 附录：全面考量清单

## A. 首批参与者需知

| 项目 | 内容 |
|------|------|
| 你加入什么 | 一个宪政约束下的协作系统，不是一家公司 |
| 你获得什么 | 分润权（按贡献、批次、规则），非股权 |

| 风险 | 程度 | 应对 |

| 机制复杂 | 中 | Phase 1 只做实核心：签约、分配、Demo |
| 人员不齐 | 中 | 先锁定 Founder + 2 IT + 2–3 艺术家 + 1 履约方 |
| 跨境法律 | 中 | 双实体 + 协议约定管辖与仲裁 |
| 信任不足 | 中 | 讨论广场、公开签约、可追溯 |

| 维度 | 标准 |
|------|------|

| 财务 | 「这东西挺公正」 |

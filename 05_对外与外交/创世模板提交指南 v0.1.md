## 1. 什么是「一个有效的英文签名模板」
在 signa.ink：

**一个英文签名模板 = 一个可被无限次调用、可被结算的独立数字资产。**

一个有效模板，必须满足以下条件：

### 1.1 基本定义
+ 对应 **一个完整的英文名字符串**
    - 例：`James`, `Alexander`, `Elizabeth`
+ 不接受：
    - 单个字母
    - 仅姓氏（Phase 2 可能开放）
    - 非拉丁字母

## 3. 模板创作原则（90 / 10 协议在英文签名中的体现）
### 3.1 90%：结构秩序（Order）
模板必须：

+ 可读性清晰（母语使用者能一眼识别）
+ 字母连接逻辑稳定
+ 无依赖上下文的“断笔谜题”

**允许：**

+ 连笔
+ 简化
+ 抽象结构

**不建议：**

+ 极端破坏字母形态
+ 仅艺术感、不可辨认的符号化处理

---

### 3.2 10%：作者笔触（Flow）
这是模板的价值来源：

+ 起笔方式
+ 转折节奏
+ 收笔态度
+ 连写时的个人惯性

⚠️ **平台不会、也禁止算法“平滑”你的笔触**

## 5. 文件格式与交付要求（v0.1）
### 5.1 必须文件
每一个名字模板，需提交：

1. **SVG（主文件）**
    - 矢量路径
    - 无外框
    - 单色（黑）
2. **PNG（预览）**
    - 白底
    - 长边 ≥ 2000px

---

### 5.2 命名规范（非常重要）
```plain
[ArtistCode]_[Name]_v1.svg
```

示例：

```plain
ZSY_James_v1.svg
```

## 7. 模板如何被“调用”（你关心的现实问题）
当发生以下行为之一：

+ 用户选择该名字模板
+ 模板用于实体印刷 / 雕刻
+ 模板进入商业协议或衍生产品

→ **即触发一次调用记录**

→ **即产生一次版税计算**

# signa.ink 基础设施与数据规划

**用途**：明确该站点的数据范围、接入节奏、与共享系统的对接方式  
**原则**：遵循《实例层架构原则》；范围与节奏可控

---

## 一、当前状态

| 项目 | 说明 |
|------|------|
| 站点定位 | 第一殖民地；签名品类 |
| 与共享库 | 待明确：当前独立库 vs 未来迁入共享库的路径 |
| 数据范围 | 待 Founder/CTO 划定 |

---

## 二、待填项（Founder/CTO 决策）

1. **数据可见范围**：哪些角色可见哪些数据；跨站点数据是否可见
2. **接入节奏**：何时接入共享库（若有）；分阶段还是一次性
3. **API 开放范围**：对外暴露哪些接口；多站点互通的范围
4. **多租户边界**：若共享库，signa.ink 的数据隔离策略

---

## 三、与其他站点的关系

- 未来 Qianming、Huahua 等站点与 signa.ink 共享同一套底层逻辑与数据库
- 本规划将随共享库方案落地而更新

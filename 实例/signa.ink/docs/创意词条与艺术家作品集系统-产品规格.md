# 创意词条与艺术家作品集系统 · 产品规格

**用途**：将创意词条（如签名级文本）结构化存储、前端展示、艺术家后台关联作品，形成可检索、可管理的资产库。  
**来源**：2025-02-10 讨论沉淀。

---

## 一、内容形态示例：A 类｜签名级文本

| 字段 | 示例 |
|------|------|
| **分类** | A 类｜签名级文本（Signature Grade） |
| **用途** | 作品落款 / 画角 / 私人物件 |
| **特点** | 主权最强、字数最少、线条最好看 |
| **词条** | Still Becoming / Quiet Confidence / Rooted & Free ... |
| **规则** | 永远不加解释文字；只出现英文；放在「不会被第一眼看到，但会被发现」的位置 |

## 二、理想状态：三端协作

### 2.1 前端（公开/半公开）

| 功能 | 说明 |
|------|------|
| **浏览** | 用户可查看创意词条、素材、分类 |
| **使用** | 选词条 → 选模板 → 下单/定制 |
| **展示** | 平台展示与词条对应的艺术家作品（示例、参考） |

### 2.2 艺术家后台

| 功能 | 说明 |
|------|------|
| **词条列表** | 看到所有创意词条/分类 |
| **上传作品** | 针对某一词条，上传自己的原版作品（对应到该词条下） |
| **作品集管理** | 知道自己哪些词条已做过、做了多少、便于管理作品集 |
| **避免重复** | 一目了然：哪些词条已有自己的作品、哪些待做 |

### 2.3 平台侧

| 功能 | 说明 |
|------|------|
| **词条管理** | 随时新增、编辑、组织创意词条 |
| **展示逻辑** | 某词条下有若干艺术家作品 → 前端展示 |
| **数据关联** | 词条 ↔ 艺术家 ↔ 作品，可检索、可统计 |

## 三、数据结构（示意）

```
创意词条（Phrase/Creative）
├── id
├── 分类（A类/签名级、B类/…）
├── 英文
├── 中文意译
├── 使用建议（星级、规则）
└── 关联作品（ArtistWork[]）

艺术家作品（ArtistWork）
├── id
├── 艺术家 id
├── 词条 id（对应哪个创意）
├── 原版文件（图/矢量）
├── 上传时间
└── 状态
```

## 四、使用场景

| 角色 | 场景 |
|------|------|
| **Founder** | 在灵感文档中整理好词条 → 批量导入平台 |
| **定义者** | 前端浏览词条 → 选模板 → 下单 |
| **艺术家** | 后台看到词条 → 为某词条上传作品 → 管理自己的作品集 |
| **平台** | 展示「某词条 + 多位艺术家作品」→ 供选择、参考 |

## 五、与现有系统关系

| 现有 | 关联 |
|------|------|
| 模板（Template） | 词条可作为模板的「语义标签」或「风格标签」 |
| 创世艺术家 | 上传作品者须为签约艺术家 |
| signa-website | 前端需有词条浏览、作品展示模块；艺术家需独立后台或扩展现有 |

---

## 六、Phase 建议

| 阶段 | 动作 |
|------|------|
| **Phase 1** | 词条以 Markdown/JSON 管理，人工维护；艺术家作品暂用外部链接或简单表 |
| **Phase 2** | 词条入库，前端可展示；艺术家后台可上传、关联 |
| **Phase 3** | 词条 ↔ 作品 ↔ 订单 全链路打通，数据可统计、可推荐 |

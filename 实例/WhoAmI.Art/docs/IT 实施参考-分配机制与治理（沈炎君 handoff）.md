# IT 实施参考：分配机制与治理

**用途**：在合适时机交给沈炎君讨论；若可行，尽快纳入系统、功能上线。  
**来源**：讨论沉淀 §9、§10；创始人全面通盘思考后的整理。

---

## 一、核心原则

| 原则 | 内容 |
|------|------|
| 无人持股 | 经济实体内部无人持有股权/股份 |
| 按贡献分润 | 合同约定，非股权；贡献、投资、参与均有回报路径 |
| 线上协议 | 任何协议均可追溯、可核查 |
| 成功标准 | IT 说「这的确挺公平」+ 财务说「这东西挺公正」 |

---

## 二、Phase 1 分配机制参考

| 要点 | 内容 |
|------|------|
| 时机 | 2 人时可讨论，3 人时再讨论；不等人齐 |
| 机制 | **逆向投票**（每人说出真实需求）+ **匹配/回流机制**（找交集、形成共识） |
| IT 参考 | 实现时，分配逻辑需参考本机制 |

---

## 三、沈炎君利益保障

| 层级 | 内容 |
|------|------|
| 协议 | 贡献类型、回报方式、比例、结算周期 |
| 执行 | 收入 → 分成计算 → 银行转账至个人账户 |
| 技术杠杆 | 掌握代码 = 核心人物；利益难被随意放弃 |
| 暂不持股 | 合同分润；协议 + 技术杠杆 = 保障 |

---

## 四、分配机制科学设计

| 维度 | 要点 |
|------|------|
| 成本/毛利/贡献 | 参考真实财务世界；非个人决定 |
| 可自动化 | 订单关联分成、固定比例、结算 |
| 需投票/仲裁 | 贡献估值争议、规则修订、异常处理 |
| 逆向表达 | 每人说需求 → 匹配机制 → 共识 → 阻力更低 |
| **贡献参数** | 经济、时间、创意或其他形式；均须有标准 |
| **回报参数** | 亦须有标准；每人清晰认知贡献与回报 |
| **系统参考** | 投票/讨论时，系统给出贡献-回报比例平均值等参考 |
| **透明** | 数据在面前，无人能赖账 |
| **可替代性** | 无人不可替代；被替换者不能无限增加功劳，防劫持 |

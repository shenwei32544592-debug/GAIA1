# WhoAmI.Art 制度补充说明（投资·分配·基础设施）

**用途**：本轮对话沉淀；投资、分配、基础设施的制度化补充  
**版本**：v1.0  
**日期**：2025-02-12

---

## 一、投资准入与分润分离

### 1.1 三条线

| 线 | 内容 | 性质 |
|----|------|------|
| **投资款** | 投资人投入，用于建设 | 资本，有回报预期 |
| **建设报酬** | IT、基础设施等劳动对价 | 工作奖励 / 未来可期回报 |
| **运营分润** | 订单产生的 Artist / Anchor / Platform 分配 | 业务分成，与投资款分离 |

### 1.2 投资准入

- 平台可接受独立投资人
- 是否接受、接受多少由平台决定
- **用途**：投资款主要用于建设传递（IT、基础设施、合规）
- **扩张**：系统扩张时可再引入财务支持

### 1.3 建设者报酬

| 情况 | 报酬方式 |
|------|----------|
| 建设期、不可替代 | 未来可期回报（延期分润、期权式激励） |
| 可被第三方替代 | 仅工作奖励（固定酬劳、项目费），**不给**长期分润份额 |

### 1.4 原则

> 投资款 ≠ 分润权；投资款进建设，分润来自运营净池；投资回报从约定结构提取，与运营分润权分离核算。

## 二、资本流与货物流分离

| 端 | 内容 | 原则 |
|----|------|------|
| **投资端 / 资本流** | 融资、分润权、FCR、跨境结算 | 独立设计与核算 |
| **运营端 / 货物流** | 订单、履约、仓储、物流 | 独立设计与核算 |

**目的**：资本变动不卡货，货流变动不卡钱；财务、物流、法律以第三方为主，按市场价采购。

---

## 三、核心 vs 非核心资源

| 类型 | 内容 | 策略 |
|------|------|------|
| **核心** | 供应端（艺术家、作品）、IT、Founder | 内建，保持强耦合 |
| **非核心** | 财务、物流、法律、营销执行 | 优先第三方，资金到位后用资金激励 |

**原则**：核心做强、做紧；非核心外包、按需付费。可替代角色不享长期分润份额。

## 四、Founder 利益与平台扩张平衡

### 4.1 无人攫取过度

| 角色 | 原则 |
|------|------|
| Founder | 从公共运营池取酬，但不独占；须预留扩张空间 |
| 早期参与者 | 有 Cap，不锁死未来增长 |
| 后续加入者 | 有清晰路径 |

### 4.2 公共运营池（20%）用途

**财政结构原则**：20% 不是「谁分走了」，而是联盟运转必须。取之于民，用之于民。

| 用途 | 说明 |
|------|------|
| 运营成本 | IT、营销、合规 |
| Founder 报酬 | 从公共运营池领取 |
| 扩张储备 | 新角色、新模块 |
| 缓冲 | Catch-up 回拨等 |

### 4.3 原则

> 各方约定：无人攫取过度。公共运营池须预留扩张与再投资空间；分配比例变更须记录、可追溯。

## 五、分配数学与扩展逻辑

### 5.1 守恒与边界

- 65% + 5% + 20% + 10% = 100%
- 各比例落在可接受区间内（Artist 60–75%，Platform 15–25% 等）

### 5.2 扩展逻辑

| 变量 | 关系 |
|------|------|
| GMV ↑ | 各方绝对收益按比例上升 |
| Anchor | 有 Cap，不无限膨胀 |
| Platform | 须覆盖随规模上升的运营成本 |

### 5.3 风险储备金（10%，FCR）说明

**财政结构原则**：10% 为全体系保险池，取之于民，用之于民——保护艺术家与用户双方。

| 用途 | 说明 |
|------|------|
| 退款/争议 | 用户退款、拒付、chargeback |
| 延迟/瑕疵 | 交付超时、质量赔付 |
| FX Buffer（1%） | 汇率波动、跨境垫付 |

**触发**：有争议时优先从风险储备金扣款赔付；不足时按规则从公共运营池或 Escrow 补足。

## 六、基础设施清单与费用

### 6.1 清单

| 类别 | 组件 | 起步期形态 |
|------|------|------------|
| **技术** | 身份、支付、订单、分配、治理日志、托管 | 手工 + 表格 或 轻量系统 |
| **物理** | 仓库/履约 | 3PL，Phase 1 纯数字可暂不用 |
| **财务** | 收款、对账、税务 | 外包专业公司，按市场价 |
| **法律** | 合同、KYC、争议、顾问 | 按需 |
| **其他** | 营销、沟通、支持 | 轻量工具 |

### 6.2 起步期月费（$）

| 场景 | 月费 |
|------|------|
| 极简 | 100–300 |
| 常规 | 300–800 |
| 稳健 | 500–1,500 |

**来源**：公共运营池覆盖。

## 七、早期参与者分配框架

| 角色 | 保障方式 | 原则 |
|------|----------|------|
| Anchor Artist | 65% + 5% Anchor Pool + Kill Switch | 已设计 |
| IT / 技术 | 固定 + 可选分润（从公共运营池切出） | 先保证能活 |
| 营销 / 运营 | 固定 + 业绩挂钩（A 类贡献） | 与订单直接挂钩 |
| 其他协作者 | B 类贡献 + 延迟兑现 | 有封顶 |

**原则**：早期保障以「足够有吸引力、可兑现」为下限，以「不锁死平台扩张空间」为上限。

## 八、系统储备金（积累池）宪政化

| 项 | 说明 |
|----|------|
| **名称** | Sovereign Reserve / 系统储备金 |
| **来源** | 公共运营池（20%）内部切分，约 2–3% |
| **性质** | 运营预算的一部分，非股权、非分红池 |
| **分阶段触发** | **验证期**：100% 用于系统安全与合规；**运营期**（日≥200 单）：可提取约定比例作绩效奖金；**成熟期**：用于禅让交接奖金与接班人激励 |
| **分红触发** | 日 ≥200 单且连续盈利 N 月后，方允许提取部分作为绩效奖金/分红；须记录 RuleID + Governance Log |
| **分配权** | Founder + 首位 CTO 共同动议；用途优先、分红后置；可审计 |

## 九、IT 伙伴激励（摘要）

| 机制 | 说明 |
|------|------|
| 里程碑现金 | 按 MVP 模块验收支付，不依赖订单 |
| Class A/B 贡献 | 绑定 RuleID，参与公共运营池分配 |
| 遗产分红 | 重构时原 CTO 贡献积分转化为持续分润权，不因代码替换归零 |
| 衍生业务优先权 | 履约、供应链、法务、财务、IT 承包等优先承接；须遵守最惠国 + 交叉审计；详见《衍生业务优先权与最惠国条款》 |
| 禅让预期 | 不终身在 CTO 位；创世公民 权益保留；交接奖金 |
| 技术多签 | 分配参数及核心代码须 Founder + CTO + 第三方多签发布；反劫持 |

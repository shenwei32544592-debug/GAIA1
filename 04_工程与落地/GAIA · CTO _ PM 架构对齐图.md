# 🧠 GAIA · CTO / PM 架构对齐图（One-Page）
## 一句话定义系统
**GAIA 不是一个应用，而是一套“源头确权 → 共识裁定 → 现实落地 → 价值回流 → 历史存证”的协议型系统。**

任何功能、任何性能优化、任何商业调整  
**都不得破坏这个闭环。**

---

## 一、系统分层总览（从上到下）
```plain
┌────────────────────────────┐
│  Alliance & Evolution Layer │  ← 跨平台 / 未来艺术联盟
├────────────────────────────┤
│  Consensus & Governance     │  ← 公众指认 / Steward 陪审
├────────────────────────────┤
│  Asset & Truth Layer        │  ← 主权资产 / 源头锚定
├────────────────────────────┤
│  Identity Layer             │  ← Node_ID / 能力掩码
├────────────────────────────┤
│  Physical Reality Layer     │  ← 工业 / 物理责任
├────────────────────────────┤
│  Atomic Ledger              │  ← 原子化分润 / 回溯补偿
└────────────────────────────┘
```

**理解顺序非常重要：**

+ 不是“用户 → 下单 → 支付”
+ 而是 **身份 → 真理 → 共识 → 现实 → 回馈 → 历史**

---

## 二、六大核心模块（工程职责版）
### 1️⃣ Identity Layer（身份层）
**解决什么问题：**

+ 系统里“谁是谁”
+ 权力从哪里来，什么时候失效

**关键设计约束：**

+ 所有参与者 = Node
+ 不使用 role enum，只用 **Capability Mask**
+ 身份锚点不可删除，只能追加

**CTO 要点：**

这是所有权限系统的根，**不可重构为 RBAC 简化版**

---

### 2️⃣ Asset & Truth Layer（资产与真理）
**解决什么问题：**

+ 什么算“原创”
+ 源头如何被长期识别

**核心对象：**

+ Sovereign Asset（主权资产）
+ Origin Anchor（源头锚点）
+ Historical Attribution（历史署名）

**PM 要点：**

这是“历史系统”，不是商品系统  
UI 可以变，但数据语义不能变

---

### 3️⃣ Consensus & Governance（共识裁定）
**解决什么问题：**

+ 谁来判断“真正的原创者”
+ 如何避免平台裁决、法律裁决

**机制要点：**

+ Public Nomination（公众指认）
+ Steward Jury（临时陪审）
+ 时间 + 专业权重

**CTO 要点：**

这是一个 **异步、低频、高权重流程**  
不追求实时，只追求可追溯

---

### 4️⃣ Physical Reality Layer（现实落地）
**解决什么问题：**

+ 防止系统沦为“纯数字叙事”
+ 建立物理责任

**核心逻辑：**

+ 所有资产最终必须可映射到现实载体
+ 存在“责任签署人”

**PM 要点：**

没有现实接口的功能 = 非核心功能

---

### 5️⃣ Atomic Ledger（收益与反哺）
**解决什么问题：**

+ 使用即反哺
+ 源头持续获益

**设计原则：**

+ 不存在“平台中转余额”
+ 每笔价值 = 多目标自动分流
+ 支持回溯补偿（Retroactive）

**CTO 要点：**

财务是协议的一部分，不是外接模块

---

### 6️⃣ Alliance & Evolution（联盟与演进）
**解决什么问题：**

+ 如何接入其他艺术领域
+ 如何并入全球慈善艺术联盟

**工程约束：**

+ ID 必须跨平台可识别
+ Schema 必须语义自解释
+ 模块必须可插拔

**CTO 要点：**

不做“封闭最优”，要做“可接入最优”

---

## 三、哪些是“不可动母结构”（红线）
CTO / PM 必须共同认知的 **四条红线**：

1. ❌ 不能删除或覆盖源头署名
2. ❌ 不能让平台成为最终裁决者
3. ❌ 不能把分润逻辑变成运营策略
4. ❌ 不能为了效率取消共识流程

**一旦违反，系统不再是 GAIA**

## 四、Sprint 视角下的理解方式

+ **Sprint 2–3：** Consensus（正义）
+ **Sprint 4–5：** Ledger + Physical（现实）
+ **Sprint 6+：** Alliance（文明）

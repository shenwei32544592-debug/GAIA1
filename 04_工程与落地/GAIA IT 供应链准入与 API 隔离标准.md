# GAIA IT 供应链准入与 API 隔离标准

**用途**：确保外部 IT 供应商只能执行任务，拿不到核心逻辑；实现**影子控制**；供采购决策与合同起草参考  
**版本**：草案 v0.1  
**性质**：工程规范，与《基建期艺术家共投机制》《分润与财务逻辑补充》配套  
**设计意图**：核心自己做、外围采购；采购而不失控；用基建池资金换取零时差扩容

---

## 一、战略逻辑：从手工坊到总承包商

当系统进入扩张期，内部技术团队（Batch 0）应从「写代码的人」进化为**规则的制定者与审计师**。

| 层级 | 策略 | 内容 |
|------|------|------|
| **Core（核心）** | 自己做 | RuleID 核心算法、分账逻辑的审计、宪章级数据接口、分配引擎——系统的「大脑」 |
| **Edge（外围）** | 直接买 | UI/UX、多语言模块、全球物流 API、云安全、CRM、支付对接——系统的「肌肉」 |

**原则**：自己研发是重新发明轮子；采购是直接驾驶跑车。采购将**固定成本（人力）**转为**可变成本（按需支付）**，保持轻资产韧性。

## 二、准入标准（黑盒采购协议）

### 2.1 API 主权化

| 要求 | 说明 |
|------|------|
| **标准化接口** | 所有外部 IT 供应必须通过联盟定义的**标准化接口**接入 |
| **供应商可替换** | 无论采购哪家服务，均为 RuleID 搬砖；某家表现不佳，可如更换电池般切换 |
| **逻辑归 GAIA** | 业务流程、分配规则、治理逻辑的**定义权与审计权**归联盟 |
| **数据归 GAIA** | 订单、分润、贡献记录等数据的**所有权与可携权**归联盟 |
| **代码归 GAIA** | 深度定制时，合同中须约定：**逻辑归 GAIA，代码归 GAIA，数据归 GAIA**；外部团队为「数字雇佣军」 |

### 2.2 源代码审计权

对于深度定制的 IT 采购，合同须明确：

- 联盟享有**源代码审计权**；可验证逻辑与接口是否符合协议
- 核心算法与分账逻辑不得外包至不可审计的黑箱
- 供应商不得将联盟数据、规则或接口用于联盟授权范围外之用途

### 2.3 切换权与退出条款

| 条款 | 约定 |
|------|------|
| **通知期** | 联盟可提前 N 日通知终止合作；供应商须配合数据迁移与接口交接 |
| **数据导出** | 终止时，供应商须提供标准格式的完整数据导出 |
| **无锁定** | 避免长期独家绑定；优先选用可替换的模块化方案 |

## 三、采购优先顺序（基建池资金用途）

当艺术家分润让渡形成的基建池有结余时，IT 采购可按下述优先顺序部署：

| 优先级 | 类型 | 说明 |
|--------|------|------|
| **P0** | 消灭交付瓶颈 | 订单积压、支付失败会透支共识信用；优先解决响应与清算 |
| **P1** | 成熟度套利 | 选用经全球验证的 SaaS、支付、云方案；避免重复造轮子 |
| **P2** | 按需扩容 | 需求激增时，采购可**今天签约、下周上线**；猎头招人需数月 |

**资金链路**：基建池（艺术家共投）→ IT 采购 → 零时差扩容 → 溢价与变现能力覆盖采购成本。

## 四、采购 vs 自研：决策参考

| 维度 | 自研 | 采购 |
|------|------|------|
| **交付速度** | 数月 | 数日至数周 |
| **成本结构** | 固定（人力） | 可变（按需支付） |
| **风险** | 招人难、上手慢 | 供应商可替换 |
| **适用场景** | 核心算法、RuleID、分账审计 | UI、翻译、物流 API、支付、CRM |
| **主权** | 完全自有 | 须通过 API 隔离与合同锁定主权 |

**原则**：核心不外包；外围不重造。

## 五、与现有协议的关系

| 协议 | 与本标准关系 |
|------|--------------|
| **基建期艺术家共投机制** | 基建池用途包含「IT 基础设施」；本标准的采购对象为该用途的落地方式 |
| **分润与财务逻辑补充** | 20% 公共运营池覆盖运营成本；IT 采购可来自基建池或运营池结余 |
| **创意载体规范** | 采购的 IT 系统须服从创意载体规范关于工业基底、自由源头等约束 |

---

## 六、资源调度中心定位

GAIA 不养数万程序员，而拥有一套精干的「制宪技术团队」+ 调动全球 IT 供应链的**财力与接口**。

竞争维度不是「谁员工多」，而是**资源调用效率**。

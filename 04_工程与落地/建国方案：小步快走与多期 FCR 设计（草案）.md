# 建国方案：小步快走与多期 FCR 设计（草案）

**用途**：以造血循环周期为锚点，延后建国锁定期，多期配置 FCR 结构；降低启动阻力，以有理有据的逻辑数据论证「这事能成」  
**性质**：战略草案；与《联盟胚胎发展》《有效路径与时间节点》《FCR 附录》配套  
**依据**：《联盟胚胎发展》§0.4 简化造血路径、《基建期艺术家共投》《建国者共建金》§5.1

---

## 一、模型目的：有理有据地说服

数据模型的首要用途：用可验证的逻辑和数字，向参与方证明「这事能成」。

| 说服对象 | 核心疑问 | 模型能提供 |
|----------|----------|------------|
| **艺术家** | 我加入后能持续赚到钱吗？ | 10×50×100 → 月 150 万 Net Pool，人均 9.75 万/月；成功信号后收益再借入可加速达标 |
| **供应商 / 协作伙伴** | 投入会不会打水漂？ | 造血临界 37.5 万/月；互信锁定后愿借出收益支持发展 |
| **投资人 / 建国者** | 需要多少启动资金、何时回本？ | 胚胎 5–15 万、婴儿 15–30 万；约 9 个月可达造血临界 |
| **Founder** | 路径可执行吗？ | 小步快走 + 人工记账 + 多期 FCR → 降低 Day 90 硬门效应 |

## 二、现行方案的结构性约束

### 2.1 单点门控

```
协议冻结 → 001 物理 → 论坛 → 首份 FCR → Day 90 建国成功投票 → 平台上线 → 接单
                                              ↑
                                         唯一硬门
```

建国成功 = 投票通过 + 至少 5 人参与 + 论坛就绪 + 首份 FCR 签署。  
在 Day 90 前：无订单、无收入、无可验证的成功信号。  
**结果**：必须在无现金流验证的情况下完成制度与社群建设，启动阻力高。

### 2.2 单一 FCR 结构

90 日窗口内所有人进入同一套 FCR；无法按阶段、造血进度、贡献类型分层。  
早期（Day 30）与晚期（Day 85）参与者风险差异大，却共用同一权益结构。

### 2.3 与造血节奏脱节

建国成功在 Day 90，平台上线在建国之后；首单、造血临界、收入爬坡均在建国之后。  
建国标准以「制度完备」为主，与「造血可期」的时间轴脱节。

## 三、延后建国锁定期 + 多期 FCR：设计思路

### 3.1 核心原则

| 原则 | 含义 |
|------|------|
| **小步快走** | 每步可验证；不设一次性大考 |
| **造血驱动** | 阶段划分以订单量、收入、现金流为主 |
| **多期 FCR** | 不同阶段配置不同 FCR 结构；与风险、贡献匹配 |
| **锁定期延后** | 正式建国锁定放在造血临界附近，非 Day 90 |

### 3.2 期与 FCR 结构示意

| 期 | 触发条件 | 产出 | FCR 结构 | 建国锁定 |
|----|----------|------|----------|----------|
| **期 0 · 胚胎** | Day 0 启动 | 协议冻结、001 物证、最小存在 | 无 FCR；意向/协议草签 | 否 |
| **期 1 · 首单** | 协议 + 001 就绪 | 能接单、能收钱、人工记账 | FCR-1：简化比例；3–5 人；贡献可追溯 | 否 |
| **期 2 · 爬坡** | 平台 50–150 单/天 | 流水可验证；互信形成 | FCR-2：扩大池；可追加早期贡献者 | 否 |
| **期 3 · 造血临界** | 平台 ~500 单/天 | 月收入 ~37.5 万；造血在望 | FCR-3：完整 65/5/20/10；Batch 0 锁定 | **是** |
| **期 4 · 成人** | 造血稳定 | 制度自动运转；可持续扩张 | FCR 分配表冻结；Batch 1+ 启动 | 已锁定 |

### 3.3 多期 FCR 设计要点

| 期 | FCR 特点 | 设计理由 |
|----|----------|----------|
| **FCR-1** | 简化比例、上限、书面约定 | 风险最高、无收入；参与者少；需轻量、可逆 |
| **FCR-2** | 扩大池、可追加贡献者 | 首单验证、互信建立；可纳入更多早期贡献 |
| **FCR-3** | 完整结构、Batch 0 锁定 | 造血临界前，贡献可量化；锁定创始权益；启动建国回补 |

### 3.4 与 90 日窗口的关系

90 日为**参考周期**，非刚性边界。  
**建国成功**以**原则性、可协商**的多维条件为实质触发（供给分散、订单可验证、可分配利润、供给集中度、结构就绪）；参考区间可协商；条件模糊或协商不一致时，由 Founder 综合判断并最终裁决。*详见《多期 FCR 制度设计》§4.1–4.2、《建国成功判定速查表》§二*

**建国时间与运转脱钩**：建国成功的长短与联盟能否正常运转无关；建国是制度确认时刻，非运转前提。美国建国早、世界领袖地位百年后；建国 ≠ 完全成熟。

### 3.5 重要节点优先认购发展基建基金

**优先接受**重要节点人（艺术家、供应商、类帐户等）对**发展基建基金**的投资认购，可使合作机制发展更平稳、更快速。

| 节点类型 | 与机制的关系 | 优先理由 |
|----------|--------------|----------|
| **艺术家** | 源头贡献者；分润主体 | 利益与平台长期绑定；已有共投机制 |
| **供应商** | 供应链协作方 | 订单增长直接利好；愿接受账期/预借换取长期合作 |
| **类帐户** | 机构、基金、合规主体 | 可规模化认购；与自然人艺术家互补 |

**原则**：重要节点认购优先于泛外部资本；其投入与系统成功高度相关，协调成本低，互信易锁定。  
*配套：《基建期艺术家共投机制》；《建国者共建金》§5.1*

### 3.6 结构性稳定，非制度性稳定

在此机制内，配合方**皆知无人可跳单独干**；利益流、分润、基建池、供应链彼此咬合，**天然形成生死同盟**。背叛无利可图，互踩无意义——这是**结构性稳定**，而非依赖规则惩戒的**制度性稳定**。

| 维度 | 制度性稳定 | 结构性稳定 |
|------|------------|------------|
| **来源** | 合约、惩戒、裁定 | 收益结构、相互依赖 |
| **背叛成本** | 违约后受罚 | 背叛即自损；离场即归零 |
| **维持方式** | 需执行、监督 | 结构自带；无需额外执行 |

**含义**：制度可简；结构一旦形成，合作无需靠「防小人」条款维持。

## 四、宪法与协议一致性

### 4.1 宪法层

- **源头不可抹除**：多期 FCR 不改变创始贡献者历史地位；仅调整确权时点与结构  
- **正义优先于效率**：提前造血不削弱透明、公平、可追溯；书面协议 + 可审计仍为底线  
- **共识高于裁决**：建国锁定仍由投票决定；仅触发条件从「Day 90」改为「造血临界 + 投票」

### 4.2 FCR 附录 §1.6

已约定 90 日非绝对刚性，以建国成功为实质锚点。  
可扩展为：建国成功 = **造血临界达成 + 投票通过**；时间窗口为辅助条件。

### 4.3 创世者权利附录

创世者署名权、经济补偿权、治理权临时性均不变。  
多期 FCR 仅调整**何时、如何**锁定分配比例，不改变权利性质。

### 4.4 需澄清事项

- **FCR 稀释**：多期加入者会摊薄既有比例；须约定「剩余空间」或「独立池」，避免事后争议  
- **窗口关闭语义**：可保留「90 日窗口」用语，但明确定义为「首期简化 FCR 的邀请窗口」，与正式建国锁定分离

### 4.5 开放 FCR + 增量约束（扩展方案）

当经共识采纳时，FCR 窗口可**不关闭**，分配可提前开启；后来者分走的须**小于**其带来的增量（T < ΔS），则既有持有者绝对额只增不减。只有不断有新血输入，盘子才能做大。*详见《多期 FCR 制度设计》附录 C*

## 五、路径对比

| 维度 | 现行路径 | 小步快走路径 |
|------|----------|--------------|
| **建国锁定** | Day 90 投票 | 造血临界 + 投票 |
| **首单时点** | 建国成功后 | 协议 + 001 就绪后 |
| **FCR 结构** | 单一 | 多期（FCR-1/2/3） |
| **启动阻力** | 高（须先完成制度） | 低（先接单验证） |
| **说服力** | 制度完备，无数据 | 有数据、有逻辑、可验证 |

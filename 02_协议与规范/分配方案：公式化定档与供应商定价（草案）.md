# 分配方案：公式化定档与供应商定价（草案）

**用途**：基于压力测试结论，设计全盘利益平衡的分配方案；FCR 贡献定档公式化、供应商可评估成本定价、裁决与异议通道、供给集中度硬线  
**性质**：协议草案；须经共识程序通过后生效；与多期 FCR、建国成功判定配套  
**版本**：草案 v0.1

---

## 〇、设计总则

| 原则 | 含义 | 制度体现 |
|------|------|----------|
| 共识优先于裁决 | 宪法第三编：同行共识优先于单一权威 | FCR 分配以协商共识为主线；Founder 裁决仅边际兜底 |
| 正义优先于效率 | 宪法精神 | 供应商定价用可评估成本+合理利润，不以竞价压价 |
| 公式防压价 | 后期 FCR 加入者不被早期节点谈判挤压 | 贡献类型+可量化指标→定档区间；无双边谈判空间 |
| 供给分散为结构前提 | 单艺术家超 25–30% = 商业模式失效 | 建国成功必要条件；无豁免、无分层 |
| 权力可过渡 | 核心领导权过早真空导致历史失败 | Founder 裁决权为建国期兜底；随治理成熟逐步转移 |

---

## 第一条｜FCR 贡献定档公式化

### 1.1 目标

- 消除**双边谈判**带来的压价空间；
- 后期 FCR 加入者（FCR-2、FCR-3）按贡献类型与量化指标落入**公开定档区间**，无需与既有持有者博弈；
- Founder 提名权保留，但须在**区间与程序**约束内行使。

### 1.2 定档公式（示意）

```
本池内比例 = f(贡献类型, 风险时段, 可量化投入, 模块边界)
```

| 输入项 | 说明 | 取值方式 |
|--------|------|----------|
| 贡献类型 | 智力/艺术/基础设施/资金/运营/社会资本 | 按 FCR 附录 §2.2 分类；每类对应基准权重 |
| 风险时段 | FCR-1 / FCR-2 / FCR-3 | 入池时点决定；FCR-1 基准最高 |
| 可量化投入 | 工时、金额、交付物数、履约质量 | 可核查记录（论坛、合同、交付物）；权重由共识约定 |
| 模块边界 | 贡献是否对应独立可识别模块 | 是/否；有模块边界者加分 |

### 1.3 各池定档参考区间（刚性）

与《多期 FCR 制度设计》§5.2 对齐；此处**收紧为公式输入**，非"建议区间"：

| 池 | 贡献类型权重基准 | 本池内比例区间 | 单人上限 |
|----|------------------|----------------|----------|
| FCR-1 | 1.0 | 15%–40% | 40% |
| FCR-2 | 0.6 | 5%–25% | 25% |
| FCR-3 | 0.4 | 3%–15% | 15% |

**计算逻辑**：系统按输入项算出"基准分"；基准分落入某档，对应比例区间。协商仅在**区间内**微调；超出区间须书面说明理由，并经门类理事会或裁定者公平性意见。

### 1.4 防压价机制

| 环节 | 设计 |
|------|------|
| 新加入者 | 其比例由公式定档，不依赖与既有 FCR 持有者的 bilateral 谈判 |
| 既有持有者 | 无权否决新加入者的定档结果；仅可经**异议通道**（见第三条）表达并记录 |
| 书面锁定 | 比例写入《FCR 授予书》时，须注明：定档依据、贡献描述、池归属；双方签署即生效 |

**思量**：公式化使"谁拿多少"可预期、可审计；无人能通过谈判把 later 加入者的份额压下去。

## 第二条｜供应商定价：可评估成本 + 合理利润

### 2.1 原则

采用**可评估成本 + 合理利润率**，而非内部竞价。理由：

- 正义优先于效率；
- 避免压价导致供应商关系工具化、互信受损；
- 与"结构性稳定"一致：长期合作、履约质量优先于价低者得。

### 2.2 基准价公式

```
供应商基准价 = 可核查成本 × (1 + 合理利润率)
合理利润率区间：15%–25%（可经共识调整）
```

| 成本类别 | 可核查方式 | 说明 |
|----------|------------|------|
| 物料 | 发票、采购单、市场价 | 实报实销或分级 SKU 定价 |
| 加工 | 工时×工时单价、工序报价 | 工序可列明；单价可对照行业参考 |
| 物流 | 快递报价、第三方结算单 | 硬成本，穿透结算 |
| 其他 | 经约定程序认定 | 须书面记载 |

### 2.3 增量竞价（边际补充）

仅下列情形可采用**有限竞价**：

- 非核心、标准化物料（如通用包装、标准底座）；
- 新品类、新供应商首次接入时的询价参考；
- 超出基准价 ±20% 浮动时的多源比价。

竞价**不替代**基准价逻辑；竞价结果若低于"成本+合理利润"下限，须书面说明理由（如规模效应、长期合作折让），并存档。

### 2.4 供应商 FCR 贡献评估

供应商若成为 FCR 持有者（如 FCR-2、FCR-3 池中的生产节点），其贡献评估**不以报价高低为主**，而以下列为主：

| 维度 | 权重建议 | 说明 |
|------|----------|------|
| 履约质量 | 高 | 准时率、合格率、纠纷率 |
| 长期合作深度 | 中高 | 合作时长、稳定供应品类数 |
| 不可替代性 | 中 | 该品类是否为独家或准独家 |
| 报价合理性 | 参考 | 符合基准价公式即可；非"价低者得权" |

**思量**：避免供应商为争 FCR 而恶性压价；FCR 奖励的是**结构性贡献**，非价格战。

## 第三条｜裁决与异议通道

### 3.1 共识主线

FCR 分配以**协商共识**为主线：

- 贡献者与 Founder（或授权代表）在定档区间内协商；
- 达成一致后写入《FCR 授予书》，双方签署即生效；
- 共识高于裁决；裁决仅在无法达成共识时介入。

### 3.2 Founder 裁决的边界

| 情形 | 处理 |
|------|------|
| 能形成共识 | 以协商结果为准；无需裁决 |
| 条件处于模糊区 | Founder 可综合判断，决定是否发起投票或定档；须书面说明理由并存档 |
| 协商无法形成一致 | Founder 最终裁决；须书面说明理由、存入 Governance Log；**不得排除异议通道** |
| 参与不足或程序僵局 | Founder 裁量；书面说明；与 FCR 附录兜底一致 |

### 3.3 异议通道（宪政底线）

**任何 FCR 分配裁决，均须保障利害关系人的表达与记录权利。** 不得以裁决排除共识与表达权，否则违宪。

| 项目 | 约定 |
|------|------|
| 异议权 | 任一 FCR 持有者或潜在贡献者，对某次 FCR 授予有异议时，有权**书面表达**并请求**记录在案** |
| 形式 | 异议登记表：异议对象、理由、署名、日期；存入 Governance Log |
| 效力 | 异议不自动推翻裁决；但须归档，供事后复盘、制度演进与审计使用 |
| 复核触发 | 当异议达一定数量或权重（如 ≥2 名 FCR 持有者联署）时，可请求裁定者或门类理事会出具**公平性意见**；意见供参考，不替代已生效协议 |

**思量**：宪法第三编"共识即合法性"；裁决不得制造"不可异议"的封闭空间。异议通道是程序正义的底线，非可选设计。

### 3.4 与建国成功判定的关系

建国成功判定中的"协商与 Founder 最终裁决"（见《建国成功判定与窗口操作速查表》§二、§六）同样须遵守异议通道；重大决策允许书面记录异议并存档。

## 第四条｜供给集中度：商业模式健康度前提

### 4.1 硬线

**单艺术家订单占比 ≤ 25–30%** 为建国成功的**必要条件**。

| 项目 | 约定 |
|------|------|
| 含义 | 任一艺术家在平台总订单中的占比不得超过 25–30% |
| 超标 | 若出现超标，表示**供给分散失败**，商业模式结构不合格 |
| 处理 | 须先解决供给侧问题（扩大艺术家基数、分散订单来源），**而非**为头部设例外或分层规则 |
| 目标规模 | 平台设计目标为可支撑**数百至千级**艺术家；单艺术家 25–30% 为健康度红线，非"可豁免"上限 |

**思量**：若任一家天然超 30%，说明供给集中度过高，平台风险过大。此红线无豁免、无分层；达标是建国成功的前提，非可讨价还价的变量。

### 4.2 与建国成功多维条件的关系

供给集中度与《多期 FCR 制度设计》§4.1 所列其他原则（供给分散、订单可验证、可分配利润、结构就绪）并列；**全部满足**时方可发起建国成功投票。

## 第五条｜权力保留与过渡

### 5.1 Founder 裁决权的定位

Founder 裁决权为**建国期兜底机制**，非日常决策手段：

- 目的：在共识失灵、程序僵局时有人能收尾，避免系统瘫痪；
- 边界：裁量须可追溯（书面说明、存档）；涉及自身利益时须回避；不得以裁决排除异议通道。

### 5.2 权力过渡路径

| 阶段 | Founder 裁决权 | 过渡方向 |
|------|----------------|----------|
| 建国前 | 保留；用于模糊区、僵局、兜底 | — |
| 建国后 | 收窄为"对既定公式的解释权"+"极少数结构性例外" | 逐步将裁量权向裁定者、门类理事会转移 |
| 制度成熟后 | Founder 退为宪章守护者；裁量权由裁定者/理事会依程序行使 | 有序过渡，非突然抽空中心 |

**思量**：历史上核心领导权过早消失曾导致失败；保留适度兜底权力以保护项目。同时，制度成熟后须实现权力向规则的转移，避免长期人治。

### 5.3 与《创世者权利与约束附录》的关系

本方案不修改创世者治理权的**临时性**（第四条）、**自动退化**（第五条）及**与治理脱钩**（经济权≠治理权）。权力过渡是治理权层面的有序交接，不扩大量裁权。

## 第六条｜操作检查清单

| 阶段 | 动作 | 检查 |
|------|------|------|
| 启动时 | 书面约定：定档公式权重、合理利润率区间、异议程序 | 存入 Governance Log |
| FCR 授予 | 按定档公式计算；协商在区间内；超区间须书面说明+公平性意见 | 本池比例之和 ≤ 100%；单人比例不超上限 |
| 供应商定价 | 可核查成本 + 15%–25% 利润；增量竞价仅边际 | 存档；RuleID 可追溯 |
| 裁决时 | 书面说明理由；开放异议登记 | 不得排除异议通道 |
| 建国成功 | 供给集中度 ≤ 25–30% 必查 | 超标则不得发起投票；须先调整供给结构 |

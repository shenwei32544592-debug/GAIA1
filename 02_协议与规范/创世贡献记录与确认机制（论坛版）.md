# 创世贡献记录与确认机制（论坛版）

**用途**：在 FCR 90 日窗口内，IT 尚未就绪时，以公共论坛为载体记录、确认创始贡献；为《FCR 授予书》比例约定提供依据；让贡献者感到被看见、未来可期  
**版本**：1.0  
**性质**：操作指引，与 FCR 1.1 及《FCR 附录：90日窗口与价值量化说明》配套使用  
**适用范围**：协议生效后首个 90 日；参与创始贡献记录与评估的贡献者

---

## 一、原则

| 原则 | 含义 |
|------|------|
| **实时可见** | 贡献在发生当时即被记录，不依赖事后回忆或 IT 自动抓取 |
| **社会确认** | 他人附和 = 认可，形成可追溯、可确权的证据链 |
| **模糊保底** | 尽力参与者获得保底式未来；杰出贡献自然可见 |
| **书面锁定** | 90 日结束后，经加权评估，将比例写入《FCR 授予书》；一经签署即固定 |

---

## 二、论坛角色与职责

### 2.1 任务发布者（创世者）

- 在论坛发布**建"国"必要任务清单**：每一具体工作、每一块真实投入、每一投资项。**建国者共建金**（小额投入、透明示范、紧急采购）见[《建国者共建金：小额投入与紧急采购机制（草案）》](建国者共建金：小额投入与紧急采购机制（草案）.md)  
- 定义「什么是必要工作」；协调节奏；接受或修正贡献归类  
- 创世期**信息最完整、理解最深刻**；其领导权为制度设计所需（见[《签字权与法人地位承继规则》](签字权与法人地位承继规则（草案）.html)§3.4）

### 2.2 确认人

- **主确认人**：创世者（Founder）  
- **副确认人**（可选）：创世技术官或 1–2 名早期 创世公民；经创世者指定  
- 职责：认可或修正贡献归类；对贡献者自报的帖子进行**附和确认**；参与最终评估

### 2.3 贡献者

- 完成工作或参与讨论后，**及时**在论坛发帖：做了什么、对应哪项任务、讨论要点  
- 他人可见；确认人及其他贡献者可**附和**（回复、点赞、明确认可）  
- 附和 = 社会确认；将来可拿出来确权

### 2.4 评估豁免

以下对象**不参与**论坛贡献评估流程：二者角色为**流程组织者与确认人**；创世期信息最完整、理解最深刻，为**制度设计所需**（见签字权规则 §3.4）；故不参与贡献评估。

**豁免理由**：权益已锁定，不重复评估；不宜既裁判又运动员。

**钱从哪来、怎么分、支配权**（简要；完整逻辑见《分润与财务逻辑补充》《FCR 附录》）：

| 对象 | 收益来源（池子） | 分配逻辑 | 支配权 |
|------|------------------|----------|--------|
| **创世者（Founder）** | **2% 分润权**（创世分润池）；1% 公益积累金（二人各 0.5% 合成，入平台公益金池）；有现金流后工作报酬可来自治理层工资 | 创世分润池：二人各 2%，其中 0.5% 合成 1% 做公益。**不参与 FCR 分利**。Time Decay、Cap 见分润逻辑 | 对**1% 公益金**有**决定性用途决策权**；对主分配参数有**设定权** |
| **创世技术官（沈炎君）** | **2% 分润权**（创世分润池）；1% 公益积累金（二人各 0.5% 合成）；有现金流后工作报酬可来自治理层工资 | 同上。**两位创始人不参与 FCR 分利**，20% 池结余全部留给 90 日内其他创始贡献者（FCR 持有者） | **收益权**（2% 按协议拿钱）；对 1% 公益金有知情权与建议权（因各出 0.5%），无决定权 |

*完整分配公式、池子用途、参数可变更性见[《分润与财务逻辑补充》](../实例/WhoAmI.Art/docs/WhoAmI.Art%20分润与财务逻辑补充（财务工程%20v1.0）.html)§二、§八至§十一；FCR 与 创世公民 区分见[《FCR 附录》](FCR%20附录：90日窗口与价值量化说明.html)§2.3、§4.2。*

**设计说明**：二人各 **2% 分润权**；两个 0.5% 合成 **1% 平台公益积累金**。创世分润池结构为 2%+2%+1%。**2% 的正当性**：既有道义（对早期投入与风险的回报），亦因二者为联盟的**原始内核人物**，承担**必不可少的守夜人角色**——协议须其在场方能运转。**钱上平等**：各拿 2%，各出 0.5% 做公益。**1% 公益金用途的提议权与最终决策权**为创世者（Founder）**专属**；可主动提议用途亦可最终决定；此为创世者制度性影响力来源，系其核心优势。创世技术官对 1% 有知情权与建议权，无决定权；其它节点有提议权与监督权。为保障公益金用途符合联盟使命、不被滥用，创世者独家行使该决策权。有现金流后工作报酬可来自治理层工资。创世者承担商业发展、制度设计、原始资源链接接入等核心投入。创世人若自愿从个人所得中捐入公益，为个人选择，不写入协议。

## 三、论坛 structure：建"国"路线图专区与讨论机制

### 3.1 建"国"路线图专区

论坛设有**建"国"路线图专区**，承载建"国"路线图网站上的全部内容：

| 要素 | 约定 |
|------|------|
| **内容同步** | 联盟宪章、成立理由书、FAQ、分润逻辑、递归反哺、创意载体规范等——网站上的全部文件在专区可查、可链接 |
| **按条款/按文件讨论** | 每一条款、每一文件设有独立讨论 thread；任何人可就具体内容提问、表达、跟贴 |
| **自由发言** | 所有人可自由发言、提出疑问；任何人可自由理解、表达、跟贴；无事前审查 |
| **创世者（总策划）** | 创世者作为总策划的起草人，**重点回复答疑**（制度、分配、战略等）、**即时改进**；其回应构成权威解释与迭代依据 |
| **CTO（IT 核心答疑者）** | 创世技术官作为**IT 相关内容的核心答疑者**；技术、实现、系统、接口等由 CTO 重点回复；与创世者分工：制度←创世者，技术←CTO |
| **双重效应** | **留存创世者权威与公信力**——通过持续、可见的答疑与改进建立信任；**透明化历史档案**——每一讨论、每一修改可追溯，形成可查证的创世档案 |

**设计意图**：围观者亦可发言。若有人发现机制有缺陷并提出改进——**应感激**，因其帮我们修正；若机制经得起检验、认同者自然会怕错过 FCR 窗口而**马上投入**。开放讨论形成自然筛选：要么改进，要么吸引。本专区即[《公民参与与贡献记账》](公民参与与贡献记账（Public%20Participation%20%26%20Credits）.md)所称**制度广场**之落地形态。**中枢战略**：全盘数据在中枢；论坛为中枢与全部节点交互之主渠道；快速、有据的 Founder 级反馈是建国期让生态看到希望的关键（详见《文档与论坛一体化方案：制宪档案库设计》§7.3）。

**与 Governance Log 的关系**：论坛讨论与创世者回复，可择要归档至 Governance Log；重大修正、共识形成须有记录。**文档与论坛一体化、版本留证、同步机制**的技术与操作设计，见[《文档与论坛一体化方案：制宪档案库设计》](文档与论坛一体化方案：制宪档案库设计.md)。

### 3.1a 发言即贡献可见（参与度展示）

**原则**：发言即被记录；账号的参与度可见，降低加入门槛、提高留存、促进共创。潜在公民在签约前即可感受「被看见」，增强归属感与投入动力。

| 要素 | 约定 |
|------|------|
| **谁计入** | 所有发言者（含围观者）；无签约门槛 |
| **展示元素** | Phase 1：发帖数、首参与日期；Phase 2：被附和次数、附和他人次数、**签约后身份标识**（认可宪章签署者头像/徽章可区分）；Phase 3：建议被采纳（创世者标记）、**权重参数可见**（Reputation Score、C/G 贡献值等） |
| **身份可见效应** | 签约认可宪章者，论坛内头像/身份可识别；权重参数公开可查；**极大促进自发维护者与建设者涌现**——人人可见谁在参与、谁有分量，形成正向示范与秩序维护 |
| **与 FCR 关系** | 参与度为 FCR 评估的**参考输入之一**，非唯一依据；90 日内有**实质贡献**者方进入 FCR 评估；参与度≠FCR 比例；权重计算详见[《GAIA 社区早期权重（Reputation Score）计算逻辑》](GAIA%20社区早期权重（Reputation%20Score）计算逻辑.md) |
| **防灌水** | 发言本身不产生权重；仅被采纳、被验证、被引用达阈值才产生权重；附和权重低于发起；明显灌水不计入；具体细则由创世期共识约定 |
| **实现** | 先简后繁；可先做发帖数、首参与日期；随论坛成熟再增附和、采纳等维度 |

**设计意图**：让潜在公民在签约前即获得「参与被看见」的体验；怕错过而更愿投入；参与度有真实参考价值（可追溯、可审计），但不过度承诺。

### 3.1b 艺术家 × 系统需求区（直连专区）

论坛设有**【艺术家 × 系统需求区】**，为艺术家→IT 的专用直连通道。仅艺术家可发主帖；IT、策展、运营仅可回复；围观者仅可阅读。每一需求须进入「已采纳/暂缓/拒绝」三种状态之一并说明理由。详见[《艺术家与系统需求直连机制（草案）》](艺术家与系统需求直连机制（草案）.md)。

### 3.2 任务清单发布（建"国"任务专区）

- 创世者在论坛开立「建"国"任务」专区；**任务责任人空缺待认领**，节点通过认领制自发填位  
- 列出：协议起草、技术模块、合规、财务、首单闭环、首批艺术家引入等  
- 每一任务可设：责任人（可选）、截止意向、产出定义  
- 清单可随进展增补修订；保持透明。**建国期人事网（层级、任务、权利的自发涌现）**设计见[《建国期人事网：自发协作结构设计（草案）》](建国期人事网：自发协作结构设计（草案）.md)。

### 3.3 贡献自报与附和

- 贡献者完成工作或参与讨论后，发帖说明：**日期、内容、对应任务、产出物**（如有）  
- 确认人或其他贡献者回复「确认」「+1」「认可」等，形成附和  
- **有效记录**：≥1 名确认人附和，或 ≥2 名其他贡献者附和（具体门槛可经创世期共识微调）

### 3.4 盘点节奏

- 不必固定为 30 日；可按 **Sprint、协议冻结、关键里程碑** 做阶段性小结  
- 创世者或指定人对论坛记录做汇总；必要时与贡献者简短确认：「我们如此识别你的贡献」  
- 目的：及时反馈，避免「做了看不见」的不安

### 3.5 90 日结束后的最终评估

- **输入**：论坛记录、附和数量、讨论深度、任务难度、产出物、**邀请人数**（直接邀请且成为创始贡献者的人数）  
- **方式**：权重设计 + 投票（创世公民 / 门类理事会 / 裁定者，视当时就位情况）+ **AI 分析作为参考**  
- **输出**：各贡献者的比例建议；经创世者与确认人确认后，写入《FCR 授予书》  
- **原则**：AI 为辅，**人的判断与投票为最终依据**；不替代共识；邀请因素不单独设池，仅作为权重输入

## 四、模糊保底与杰出可见

### 4.1 保底

- 凡在 90 日内有**被确认人正式认可的实质性贡献**，即享有保底资格  
- 保底形式：写入《FCR 授予书》的**最低比例**，或参与 FCR 溢出回流的共享池（具体由创世期共识约定）  
- 只要尽力参与，即有不错的保底式未来

### 4.2 杰出

- 贡献大者，帖子多、附和多、讨论深，**人人可见**  
- 最终评估时，杰出贡献自然获得更高权重与比例  
- 无需事前精确公式；社会可见度 + 加权投票 + AI 参考，足够区分

---

## 五、与主协议的关系

- 本机制**不修改** FCR 1.1 的实质性条款  
- 论坛记录为**《FCR 授予书》比例约定的依据之一**；书面协议为法律效力来源  
- 若机制与 FCR 1.1 冲突，以 FCR 1.1 为准

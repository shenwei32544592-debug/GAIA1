# 签字权与法人地位承继规则（草案）

**用途**：规定联盟代表会对外签字的权力归属、产生程序、任期轮替及承继机制；确保长期以后「谁的签字有法律效力」由制度决定，而非个人指定  
**版本**：v0.1 草案  
**联盟宪章引用**：《GAIA 联盟宪章》第六编（修正、演进与文明延续）；《宪章型协作协议》第五条（禅让保护）  
**性质**：协议草案，待制宪会议与共识程序确认后写入正式文本  
**配套**：《执行节点专项授权书》《宪章型协作协议》《GAIA 技术可行性说明（公众版）》§7.8.2

---

## 第一条：目的与原则

### 1.1 目的

联盟代表会由《自然人宪章》签署人组成，非法人。其对外签署法律文件（如执行节点授权书、重大合同、协议修订确认）须经**授权签字人**行使。本规则规定：

- 谁有资格成为授权签字人
- 签字人如何产生、如何轮替
- 签字生效的条件
- 长期承继机制

### 1.2 原则

| 原则 | 含义 |
|------|------|
| **制度高于个人** | 签字权来自联盟宪章或自然人宪章规定的程序，不依附于某一人或某一公司 |
| **在任有效** | 授权签字人的签字，仅在任期内、就授权范围内事项有效；卸任即失效 |
| **可查可证** | 当前有效签字人名单须在 GAIA 官网或创世档案可查；对外方可知谁有权签字 |
| **可承继** | 承继规则写入本协议；修正须经联盟宪章第六编规定的共识流程 |

### 1.3 反劫持

**关键角色须有备份**：主签字人、创世技术官、核心运营等关键角色，须有备份或可替代方案；**单人不可挟持系统**。备份签字人、技术交接文档、权限分散等，均为反劫持设计。若关键角色失能、失联或滥用，可经本规则规定的程序启动承继或罢免。

### 1.4 无人受雇与多人协作

**无人受雇**：授权签字人为选举或指定产生的**受托人**，非执行节点或联盟雇员。联盟不与任何自然人建立雇佣关系；参见《分润与财务逻辑补充》§5.2 无人受雇原则。

**多人协作**：法定签字可采用 k-of-n 多签或按文件类型分工；具体由制宪期共识约定。多人协作不改变签字权来源于选举/指定的性质。

## 第二条：授权签字人的定义与范围

### 2.1 定义

**授权签字人**：经本规则规定程序产生、有权代表联盟代表会对外签署本规则所列法律文件的自然人。

### 2.2 签字权适用范围

| 文件类型 | 示例 |
|----------|------|
| 执行节点授权书 | 向 LLC 发出或撤回授权；指定下一执行节点 |
| 分配参数变更确认 | 多签流程中创世者之外的必要联署 |
| 重大对外合同 | 须联盟代表会背书的战略合作、品牌授权等 |
| 协议修订的联盟代表会确认 | 宪章型协作协议修订后的联盟代表会签署 |

### 2.3 不适用范围

分配引擎日常执行、单笔交易、平台运营决策——这些由 RuleID、执行节点、门类理事会按既有规则处理，不须授权签字人签署。

### 2.4 1% 公益金用途决策权

1% 公益金用途的决策权为**独立于签字权**的治理权力；当前归属创世者（Founder）专属。其**移交方式**（死亡失能时的承继、创世者自愿交出予委员会）见《公益金池细则》§4.4。签字权承继与 1% 决策权移交可同步触发，但二者分别规定。

## 第三条：创世期的授权签字人（Phase 0）

### 3.1 创世期界定

自《自然人宪章》首批签署至**首届治理机构就位**（门类理事会、裁定者具备裁定能力）为止，为创世期。

### 3.2 创世期签字人产生

| 条款 | 内容 |
|------|------|
| 3.2.1 主签字人 | 创世者（Founder）为创世期**主授权签字人**，有权代表联盟代表会签署本规则第二条所列文件 |
| 3.2.2 联署要求 | 执行节点授权书、分配参数变更等关键文件，须**主签字人 + 至少一名其他 创世公民 签署人**联署生效 |
| 3.2.3 备份签字人 | 联盟代表会可书面指定 1–2 名**备份签字人**（须为 创世公民）；主签字人不可行使时，由备份签字人按同样联署规则代行 |
| 3.2.4 存证 | 主签字人、备份签字人名单及授权范围，存入 Governance Log；官网可查 |

### 3.3 创世荣誉监护（风险解除后）

**风险解除**后（见术语表：裁定者≥3、分配引擎 24 月无人工干预、门类理事会≥3 门类），创世者与创世技术官可申请转为**创世荣誉监护（Founding Guardian）**：2% 分润权保留，治理权移交，署名永久。签字权按本规则第四条及后续条款产生的继任人行使；创世荣誉监护不再担任主签字人或备份签字人，系光荣退出的一种。

### 3.4 创世期提前结束

若门类理事会、裁定者在创世期结束前就位，可经 创世公民 过半数同意，提前启动第四条「长期规则」。

### 3.5 创世期领导权

创世期内，创世者（Founder）与创世技术官作为**主要协调者**，享有制度设计的领导权：

| 依据 | 说明 |
|------|------|
| **信息不对称** | 创世者掌握制度、法律、规划、风险、产品全貌；对系统理解最深刻；其他节点在局部视角，难以做全局判断 |
| **时间窗口** | 90 日有限；若事事共识，易延误；需有人能拍板、协调，确保制宪与首版交付 |
| **风险承担** | 决策错误、节奏延误的后果，由创世者与核心节点承担；权责一致 |
| **制度过渡** | 领导权为**创世期工具**，非永久统治；制度就位后，治理权向 创世公民、门类理事会、共识机制过渡 |

**实践含义**：大家需在创世期内听从创世者与创世技术官的指导；其发布的任务清单、贡献确认、流程组织，具有权威性。详见《创世贡献记录与确认机制（论坛版）》。

## 第四条：长期的授权签字人（Phase 1+）

### 4.1 长期规则生效时点

首届治理机构就位（门类理事会 ≥1 个、裁定者具备裁定能力）且 创世公民 过半数同意后，长期规则生效。

### 4.2 签字人产生程序

| 条款 | 内容 |
|------|------|
| 4.2.1 提名 | 主签字人由**联盟代表会（创世公民 签署人）**推举；或由**门类理事会联合创世者**提名 |
| 4.2.2 确认 | 须经**创世公民 过半数**书面同意；或经**门类理事会 + 不少于 2 名裁定者** 共识确认 |
| 4.2.3 任期 | 每届**3 年**；可连任；连任须重新经 4.2.2 确认 |
| 4.2.4 人数 | 主签字人 1 名；备份签字人 1–2 名；均须为 创世公民 或经 创世公民 与门类理事会共同认可的自然人 |

### 4.3 签字生效条件（长期）

| 文件类型 | 签字要求 |
|----------|----------|
| 执行节点授权/撤回 | 主签字人 + 至少 1 名备份签字人或 创世公民 成员 |
| 分配参数变更 | 创世者书面同意 + 多签（见《技术可行性说明》§7.2.1）；授权签字人可作联署方 |
| 重大对外合同 | 主签字人 + 门类理事会至少 1 名代表联署 |
| 协议修订确认 | 主签字人 + 创世公民 过半数书面确认 |

### 4.4 卸任与承继

| 条款 | 内容 |
|------|------|
| 4.4.1 主动卸任 | 主签字人可提前 90 天书面通知卸任；卸任前须完成承继程序 |
| 4.4.2 被动罢免 | 经 创世公民 过半数 + 门类理事会联合动议，或经裁定者共识裁定（如违宪、严重失职），可罢免主签字人 |
| 4.4.3 承继程序 | 主签字人卸任或罢免后 30 日内，按 4.2 程序产生继任；过渡期由备份签字人代行，须满足联署要求 |
| 4.4.4 空缺 | 若 30 日内未能产生继任，由门类理事会主席（或轮值代表）临时行使主签字人权，直至正式产生 |

### 4.5 死亡、失能等不可抗力

主签字人死亡、失能、失联时：备份签字人立即代行；90 日内完成正式承继程序。若备份签字人亦不可用，由 创世公民 召集紧急会议，过半数推举临时主签字人。

## 第五条：法人受托（可选，长期）

### 5.1 原则

联盟代表会为非法人联合体。若未来需以**法人身份**对外签约（如设立公益信托、基金会、专项实体），可经本规则规定的共识程序，**委托特定法人**作为联盟代表会的受托签约主体。

### 5.2 受托法人产生

| 条款 | 内容 |
|------|------|
| 5.2.1 动议 | 由门类理事会或 创世公民 过半数提议 |
| 5.2.2 确认 | 须经 创世公民 过半数 + 门类理事会联合决议；或经联盟宪章第六编修正案程序 |
| 5.2.3 约束 | 受托法人的章程、治理结构须与本规则及联盟宪章一致；联盟代表会保留撤回委托权 |
| 5.2.4 签字权 | 受托法人的对外签字人，须由联盟代表会授权的签字人指定或确认；受托法人不替代、而是延伸联盟代表会的签字能力 |

### 5.3 当前阶段

创世期及早期运营期，**不设立**受托法人；授权签字人直接代表联盟代表会签署。受托法人为长期可选架构，待治理成熟后按需启动。

## 第六条：名单公示与存证

### 6.1 公示义务

| 内容 | 存放位置 |
|------|----------|
| 当前主签字人、备份签字人名单 | GAIA 官网「创世档案」或「治理结构」板块 |
| 上任日期、任期届满日 | 同上 |
| 历届签字人及任期 | Governance Log；可摘要公示 |
| 本规则全文及修订历史 | GAIA 官网；协议与规范板块 |

### 6.2 对外方告知

与联盟代表会签约的第三方，有权要求出示「当前授权签字人」的证明；联盟代表会须提供官网链接或经签署的授权声明。

## 第七条：修正程序

本规则修正须满足：

1. 修正案不得削弱签字权的程序约束（如不得改为「创世者终身签字有效」）
2. 须经 创世公民 过半数同意 + 门类理事会确认
3. 若涉及联盟宪章衔接，须同步履行联盟宪章第六编修正程序
4. 修正案须设定不少于 90 天的缓冲期

---

## 第八条：与联盟宪章的关系

本规则为**宪章配套协议**，效力低于《GAIA 联盟宪章》，高于一般商业合同。本规则与联盟宪章冲突时，以联盟宪章为准。本规则未尽事宜，参照[《宪章型协作协议》](../05_对外与外交/WhoAmI.Art%20宪章型协作协议（自然人专用版）结构草案.html)、[《执行节点专项授权书》](../05_对外与外交/WhoAmI.Art%20执行节点（壳公司）专项授权书（摘要版）.html)及联盟宪章第六编。
